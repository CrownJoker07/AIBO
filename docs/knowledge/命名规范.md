# AIBO 知识库命名规范

> 本文档定义了知识库文档的命名标准，确保一致性和可维护性。

---

## 文档分类

### 1. 知识沉淀文件
**命名**: `知识沉淀.md`
**位置**: 每个分类根目录
**说明**: 汇总该分类下的核心洞察、统计和索引
**示例**: `docs/knowledge/商业化/知识沉淀.md`

### 2. 索引文件
**命名**: `index.md`
**位置**: 知识库根目录和各分类目录
**说明**: 提供快速导航和内容概览
**示例**: `docs/knowledge/index.md`

### 3. 自动学习的文档
**命名**: `YYYY-MM-DD-Article-Title.md`
**说明**:
- 使用原始内容发布日期
- 如果无法确定发布日期，使用获取日期
- 标题应简洁、描述性

**示例**:
```
2026-03-18-Lostgarden善意游戏设计完整指南.md
2025-01-22-Are-Better-Models-Better.md
```

### 4. 手动创建的文档
**命名**: `YYYY-MM-DD-Descriptive-Title.md`
**说明**: 手动创建的原创分析、见解或汇总
**示例**:
```
2026-03-21-游戏行业2026年趋势分析.md
2026-03-20-AIBO原创观点-混合变现策略.md
```

### 5. GDC/会议演讲文档
**命名**: `YYYY-Talk-Title.md`
**说明**: 保留原始会议标识符格式
**示例**:
```
2024-The-Many-Dimensions-of-Pixel-Physics.md
2025-Eternal-Strands-音乐创作.md
```

---

## 命名规则

### 必须遵守
1. **日期格式**: `YYYY-MM-DD-` (4位年-2位月-2位日)
2. **标题分隔符**: 使用 `-` (连字符) 分隔标题中的单词
3. **避免特殊字符**: 标题中不要使用 `:` `?` `/` `\` `|` 等
4. **中文使用**: 中文标题直接使用中文，英文标题使用 PascalCase 或 kebab-case

### 建议遵守
1. **标题长度**: 控制在 30-60 字符
2. **描述性**: 标题应能反映文档内容
3. **一致性**: 同类文档使用相同的命名模式

---

## 迁移指南

### 批量重命名（可选）

如果需要进行批量重命名，建议：

```bash
# 示例：使用 git mv 保持历史
git mv old-name.md "2026-03-21-new-name.md"
```

### 不应重命名的文件
- `知识沉淀.md` - 分类汇总文件
- `index.md` - 索引文件
- 带有明确会议/事件标识的文件（如 GDC 演讲）

## 验证

检查命名规范执行情况：

```bash
# 查找不符合规范的文档
find docs/knowledge -name "*.md" -type f | while read f; do
  basename "$f" | grep -qE "^[0-9]{4}-[0-9]{2}-[0-9]{2}-|^知识沉淀\.md|^index\.md" || echo "$f"
done
```
