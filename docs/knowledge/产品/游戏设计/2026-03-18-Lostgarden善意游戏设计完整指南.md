# 善意游戏设计完整指南 (Kind Games: Designing for Prosocial Multiplayer)

> **文档来源**: https://lostgarden.com/2023/07/08/kind-games-designing-for-prosocial-multiplayer/
> **主页**: https://lostgarden.com/
> **作者**: Daniel Cook, JC Lau, May Ling Tan, Joel Burgess, Tomo Moriwaki, Erin Drake Kajioka
> **原始发布**: Polaris Game Design Retreat 2022 papers
> **获取日期**: 2026-03-18

## 概述

**善意游戏 (Kind Games)** 是指从一开始就设计有促进亲社会行为系统的多人游戏。本文提出了一种全新的游戏设计方法：主动设计促进积极人际关系的游戏系统，而非被动应对毒性问题。

### 核心定义

> **善意游戏** = **多人游戏 + 从一开始就设计有促进亲社会行为的系统**

### 善意游戏属性

1. 系统鼓励玩家互相帮助和帮助社区
2. 玩家形成缓解孤独的真实情感联系
3. 群体之间和平互动
4. 毒性行为受到仔细监控和调解
5. 社交系统培养归属感

## 第一章：善意游戏定义

### 问题：反应式和短视的社交系统设计

过去30年常见的两种错误做法：

| 错误做法 | 描述 | 问题 |
|----------|------|------|
| **反应式设计** | 先上线后修补 | 无法预测大规模人群的行为 |
| **复制设计** | 照搬现有系统 | 继承了旧设计的深层问题 |

### 毒性现状数据

| 游戏 | 毒性数据 |
|------|----------|
| **League of Legends** (2022) | 81% 成人玩家遭受骚扰 (2019年为75%) |
| **Valorant** | 84% 骚扰率 |
| **首次遭遇毒性** | 玩家流失率高 32 倍 |
| **总体** | 33% 玩家因骚扰退游 (2022) |

### 善意价值观体系

```
┌─────────────────────────────────────────────────────────┐
│                    善意价值观                            │
├─────────────────────────────────────────────────────────┤
│  必须有 (Must Haves)                                    │
│  ├─ 安全 (Safety)                                       │
│  └─ 相互依赖 (Interdependence)                          │
├─────────────────────────────────────────────────────────┤
│  加速器 (Accelerators)                                  │
│  ├─ 共同目标 (Shared Purpose)                           │
│  ├─ 归属感 (Belonging)                                  │
│  └─ 同理心 (Empathy)                                    │
├─────────────────────────────────────────────────────────┤
│  维护 (Maintenance)                                     │
│  ├─ 多样性 (Diversity)                                  │
│  ├─ 友谊形成 (Friendship Formation)                     │
│  └─ 健康冲突 (Healthy Conflict)                         │
└─────────────────────────────────────────────────────────┘
```

### 善意游戏案例

| 游戏 | 善意特点 |
|------|----------|
| **Sky: Children of Light** | 最佳实践的新玩家友谊培养流程 |
| **Journey** | 早期善意游戏典范，限制社交可供性消除毒性 |
| **Animal Crossing** | 精心管理多人互动减少毒性 |
| **Death Stranding** | 社交纽带系统，异步互助 |
| **Elden Ring** | "Let Me Solo Her" 社区英雄现象 |
| **FF14** | 社区导向的MMO设计 |

## 第二章：商业机会

### 为什么做善意游戏？

#### 伦理机会

- 游戏是现代社会主要社交空间
- 传统第三空间（游乐场、酒吧、俱乐部）正在消失
- 孤独感上升、社区信任下降、社会纽带衰减
- 善意游戏可以：
  - 促进健康友谊形成
  - 减少孤独感
  - 降低毒性
  - 提供目标感和归属感

#### 商业机会

| 商业逻辑 | 说明 |
|----------|------|
| **留存是成功关键** | GaaS 模式下，长期留存驱动商业成功 |
| **毒性导致流失** | 33% 玩家因骚扰退游 |
| **友谊驱动留存** | 多个朋友一起玩时留存率飙升 |
| **社区带来自然获客** | 口碑传播是主要获客渠道 |
| **品牌建设** | 善意游戏是长期品牌信任积累策略 |

### Animal Crossing 飞轮品牌案例

```
用户信任 ──→ 社区增长 ──→ 口碑传播 ──→ 新用户
    ↑                                      │
    └────────── 正向体验 ←─────────────────┘
```

## 第三章：亲社会性 (Prosociality)

### 定义

> **亲社会行为** = 一个人帮助另一个人的行为类别（与意图或奖励无关）

### 亲社会性二维模型

```
              帮助 (Helping)
                  ↑
                  │
    利他主义     │     互惠双赢
    (Altruistic) │     (Win-Win)
                  │
自我导向 ←────────┼────────→ 他人导向
(Self)           │        (Other)
                  │
    自私         │     欺骗
    (Selfish)    │     (Cheater)
                  │
                  ↓
              伤害 (Harming)
```

### 意图类型光谱

| 意图类型 | 定义 | 善意游戏关注 |
|----------|------|--------------|
| **利他主义** | 仅对接受者有益 | 理想但罕见 |
| **双赢** | 对双方都有益 | 最常见 |
| **自利** | 对自己有益，对他人中性或有害 | 避免 |
| **欺骗** | 假装利他实际自利 | 避免 |
| **事与愿违** | 意图帮助实际有害 | 避免 |
| **交易性** | 帮助他人期待回报 | 可接受 |

### 为什么不直接用"利他主义"？

利他主义要求：
- 高成本给帮助者
- 高收益给接受者
- 广泛认可为有益
- 清晰的纯粹利他意图

**这门槛太高了！** 善意游戏务实接受双赢情境。

## 第四章：游戏设计模式

### 模式1：让双赢互动变得容易

**核心原则**: 构建双赢情境，帮助他人既容易又有回报

**案例: Overwatch 的 Lucio**
- 支持角色的能力是范围效果
- 增强队友生命回复或速度
- 同时自己也受益
- 不需要做出零和牺牲

### 模式2：前置玩家承诺和行为准则

**博弈论基础**: 大多数玩家使用"以牙还牙"策略，只在预期他人合作时才合作

**设计策略**:
1. 明确声明游戏是合作空间
2. 设定积极初始期望
3. 定期强化承诺
4. 公开展示违规处理

**案例: Red Dead Redemption 2 的 Rift Trails**
- 爱好者组织的马术社区
- 明确的群体身份：热爱骑马的包容环境
- 清晰的活动预期
- 违规者会被踢出

### 模式3：牺牲机制

**零和情境中的利他主义机会**：

设计要素：
1. **显示需求** - 受助者通过游戏机制发出信号
2. **可共享资源** - 有限的、有价值的资源
3. **显示牺牲** - 帮助者展示放弃资源的动作
4. **认可牺牲** - 不要外在地奖励，但可以给社会性认可
5. **观众** - 朋友或群体内成员更可能触发利他行为

### 模式4：善意与残酷的对比

**对比增强美学**: 将利他行为与残酷背景对比，放大亲社会信号的影响

**案例: Elden Ring 的 Let Me Solo Her**
- 世界充满残酷和贪婪
- 玩家多次失败体验绝望
- 双重选择进入：金召唤标志明确表示帮助意图
- 随机性消除交易感
- 社区英雄形象展示集体价值观

### 模式5：赠礼经济

**模仿历史采集-狩猎社会的赠礼经济**：

经济设计要素：
- 玩家是小型持续群体的长期成员（约50人）
- 大多数狩猎失败或低于维持水平
- 少数狩猎大获成功（运气成分大）
- 资源有价值且会快速腐烂
- 成功是公开知识（藏不住猛犸象）
- 自私=浪费=社会惩罚

**结果**: 理性策略是赠礼给群体成员，建立互惠契约

**案例: Farmville 的赠礼系统**
- 赠礼无成本
- 赠礼获得即时奖励
- 问题：太容易导致垃圾信息

### 模式6：感知慷慨

**心理学发现**: 接受者关心的是**感知到的**意图，而非实际意图

**设计技巧**：

| 技巧 | 说明 |
|------|------|
| **新关系** | 新玩家不知道激励存在 |
| **合理推诿** | 隐藏激励，意图保持模糊 |
| **价值不对称** | 低成本给高价值物品显得慷慨 |
| **模糊不对称** | 隐藏真实财富，让赠礼显得更珍贵 |

## 第五章：反模式

### 反模式1：有毒利他主义

**问题**: 付出过多导致倦怠和流失

**预防策略**:
- 价值观对齐让工作充满能量
- 鼓励分布式支持（不要一个人承担）
- 设置适当边界
- 防止"更多就是更好"的激励

### 反模式2：稀缺与丰裕

**稀缺导致毒性的条件**：
1. 有限的零和资源来源
2. 高普遍效用 + 嫉妒
3. 分享期望被违反
4. 陌生人互动

**极端丰裕也有问题**: 玩家不再需要他人

**平衡**: 适度稀缺促进合作，过度稀缺导致毒性

### 反模式3：个人竞争

**问题**: 过度强调竞争和个体利益激励自私行为

**例外: 体育精神**
- 竞技空间内有限零和竞争
- 竞技空间外有共同身份和仪式
- 共同训练和时间
- 治愈仪式（如冰壶获胜者买输家啤酒）

### 反模式4：黑暗三联征

| 人格特质 | 特点 | 吸引的游戏系统 |
|----------|------|----------------|
| **自恋** | 认为自己最好 | 名人/地位系统 |
| **马基雅维利主义** | 目的证明手段 | 结果导向奖励 |
| **反社会** | 无悔意、反社会 | 残酷行为获益 |

**应对策略**:
- 这些人占总人口 <5%
- 权力位置特别吸引他们
- 考虑反腐败设计
- 追溯性封禁是当前最佳实践

### 反模式5：恶意破坏 (Griefing)

**定义**: 限制他人选项的敌意行为

**案例: Sea of Thieves 联盟系统**
- 原本设计为合作机制
- 联盟旗帜让其他玩家可见
- 完成任务所有人获得金币
- **被滥用**: 恶意者用联盟追踪其他玩家位置进行骚扰
- **结果**: 玩家避免使用联盟系统

**教训**:
1. 不要做天真的设计师
2. 预先推演亲社会功能如何被滥用
3. 恶意者需要容易接触非恶意者才能生存
4. 切断恶意者与受害者的联系

## 第六章：支持价值观的设计

### 安全 (Safety)

**定义**: 玩家可以分享观点和真实自我而不担心声誉损害或反击

**模式1: 积极互动的清晰信号**
- 识别建立信任的积极互动
- 通过激励或降低成本提高频率
- 自动反馈/信号
- 公开展示（kudos系统）

**模式2: 渐进式开放可供性**

Sky: Children of Light 的渐进解锁：
```
匿名影子 → 交换光揭示身份 → 赠送蜡烛 → 好友列表 → 跟随/传送 → 开放聊天
```

**模式3: 退出和回退到更安全的封闭可供性**
- 多层同意和控制
- 语音聊天失败可回退到ping系统
- 为最边缘群体设计安全选项

**反模式: 通过负面情绪生成需求**
- 社交比较驱动消费
- 算法推广情绪化内容
- 人为制造焦虑再缓解
- **思考**: 安全系统是否只是贴在毒性核心上的创可贴？

### 相互依赖 (Interdependence)

**定义**: 玩家必须依赖他人才能实现个人无法完成的目标

**反模式: 个人权力幻想**
- 西方游戏的历史依赖
- 玩家升级获得神一般的能力
- 不需要任何人就能改变世界
- 吸引社会化不良的年轻男性（16-25岁）

**模式1: 技能专业化**
- MMO的"神圣三位一体"：DPS、坦克、治疗
- Mario Party：一人踩按钮另一人通过
- DOTA：Carry、中单、劣单、游走、硬辅

**模式2: 资源专业化**
- Star Wars Galaxy：玩家拥有特定矿石，他人需要交易
- Animal Crossing：每个岛屿只产特定水果

**模式3: 软合作 "在一起更好"**
- Realm of the Mad God：敌人血量是全局属性
- 更多玩家射击=更快杀死=更少风险
- 战利品共享给所有伤害者

**模式4: 知识共享**
- 经济生产信息：稀有矿石位置
- 制度知识：最佳副本组织方法
- 培训知识：老手教新手
- 高维商品：质量不同、规则不透明的材料
- 异步知识共享：Dark Souls/Elden Ring 的留言系统

**模式5: 人类接触的稀缺与喜悦**

Journey 的设计：
- 玩家很少遇到其他人
- 遇到时产生巨大喜悦
- 封闭通信系统（单一鸣叫）限制毒性
- 无碰撞限制恶意破坏
- 能量提升是实例化的（自私行为看起来像利他的信息共享）

### 共同目标 (Shared Purpose)

**定义**: 玩家为比自己更大的事物工作

**促进因素**:
- 社会支持
- 个人成长
- 自我效能感

**模式1: 共同社区目标**
- Minecraft 服务器的大型建筑社区
- 整个社区共同创建宏伟作品

**模式2: 聚焦帮助他人**
- Destiny 2 的 Player Guides
- r/DestinySherpa：9万成员的向导社区
- 严格规则禁止自我推销或精英主义

### 归属感 (Belonging)

**定义**: 玩家在社区中感到强烈的成员感

**反模式: 他者化 (Othering)**
- 焦虑、怨恨、对"他者"的恐惧
- 自我表达如果不加管理可能无意中创造他者感
- 群体凝聚力越高，反社会行为越少，普遍信任越高

**模式1: 加入仪式**
- 新成员必须克服的挑战
- 强调价值观的宣誓
- 社区的公开认可

**模式2: 成员标记**
- 公会名称缩写
- 特定染色的服装
- 一眼识别盟友

**模式3: 投资与所有权**
- Valheim：玩家共同创建和维护基地
- Sea of Thieves：玩家聚集演奏音乐

### 同理心 (Empathy)

**定义**: 玩家寻求理解他人视角

**模式1: 具身化**
- VR 标题尝试不同身体和身份
- 提供其他媒介无法实现的生活视角

**模式2: 创造反思空间**
- Kind Words：玩家发送想法到数字海洋
- 鼓励阅读、反思、诚实回应
- 与无限滚动的注意力争夺 UI 相反

### 多样性 (Diversity)

**定义**: 玩家重视多元身份和观点

**为什么重要**:
- 刻板印象导致不真实、交易式对待
- 群体思维导致严重的毒性群体行为
- 自然倾向是相似性分组（小世界网络）

**设计策略**:
- 委派高度连接的枢纽个体（超级连接者）
- 建立与其他群体的关系
- 这些弱桥接纽带带来与不同人的接触
- 与多元化人群的个人关系减少对刻板印象的依赖

### 友谊形成 (Friendship Formation)

**定义**: 玩家形成强大、有意义的关系

**友谊光谱**: 从陌生人积极互动 → 信任、同情、支持

**模式1: 互惠关系**

Death Stranding 社交纽带系统：
- 自利动机：建造桥梁方便自己
- 自动分享：创作出现在其他玩家地图上
- 低成本互惠：点赞
- 友谊升级：看到更多朋友作品
- 进一步互惠：更多帮助和被帮助机会

**模式2: 朋友间的玩笑**

关键区别：
- 朋友间侮辱 = "我知道你知道我的意图是友好的"
- 陌生人同样侮辱 = 直接攻击

Sea of Thieves 案例：
- 2-4人小组
- 通常是现实朋友
- 恢复成本相对低
- "恶意行为" = 有趣的纽带

### 健康冲突 (Healthy Conflict)

**定义**: 冲突是不可避免的，目标是帮助健康处理

**模式1: 用封闭可供性数字化调解冲突**

MMO交易案例：
- 以前：5-10% 交易有投诉
- 解决：安全交易窗口
  - 自动验证物品
  - 双重选择进入
  - 任何更改重置确认
- 现在：<1% 投诉

**模式2: 决策系统**

必须覆盖的领域：
- **分析**: 什么数据进入决策框架？
- **权威**: 谁有权力？
- **决策机制**: 随机/投票/共识/市场/独裁
- **沟通渠道**: 讨论需要聊天，市场需要价格
- **实施**: 游戏代码执行
- **审查/反馈**: 如何知道决策成功？
- **反腐机制**: 任期限制、监督角色、半随机选择

## 第七章：善意游戏开发流程

### 园艺隐喻

```
开发者播种基础结构 ──→ 社区培育社交规范 ──→ 共同创作
```

### 开发阶段

| 阶段 | 活动 |
|------|------|
| **社交架构** | 实现社交系统基础 |
| **社区播种** | 吸引初始社区，玩家承诺和行为准则至关重要 |
| **善意度量** | 超越留存和变现，关注社交网络结构 |
| **害虫控制** | 处理极端反社会行为 |
| **修剪** | 消除或转化糟糕的设计方面 |
| **施肥** | 观察玩家行为，放大共鸣 |

### 善意度量指标

**超越留存和变现**：

1. **核心价值度量**
   - 每个关键价值的代理指标
   - 帮助他人的关键动作

2. **社交网络分析**
   - 友谊和群体分布
   - 枢纽玩家是谁
   - 参与模式
   - 检测流失或倦怠
   - 慷慨如何通过网络传播

3. **心理调查工具**
   - 动机、幸福感等量表
   - 不要害怕主观情感度量

## 附录

### 舒适 (Cozy) vs 善意 (Kind)

| 特征 | 舒适游戏 | 善意游戏 |
|------|----------|----------|
| **关注点** | 亲密和反思 | 亲社会社区行为 |
| **规模** | 个人或小团体 | 2-150人群体 |
| **空间** | 小空间 | 可扩展到大型部落 |
| **压力源** | 安全环境避免压力 | 健康调解社交压力 |
| **目标** | 创建安全空间 | 建立信任、友谊、社会资本 |

### 社交架构变量

**群体规模**：
- 个体
- 二人组
- 小组
- 大组
- 社区
- 社会

**传统控制变量**：
- 心理特征
- 动机
- 人口统计
- 通信媒介
- 网络级变量

**情境和社交架构因素**：
- 世界大小
- 持久性
- 竞争vs协作
- 角色扮演
- 沙盒vs线性
- 表现形式
- 互动可供性
- 行为成本
- 本地文化

### 亲社会行为影响因素

**联系强度**：
- 接受者是亲密朋友/亲戚？→ 更可能帮助
- 特定个人vs一般群体？→ 更可能帮助特定个人
- 内群体vs外群体？→ 更可能帮助内群体
- 近期互惠？→ 更可能回报

**群体规模**：
- Dunbar 数：约150人维持关系
- 亲密高信任关系：数量更少
- 洪水般的人=陌生人=不信任
- 人太少=孤独

**价值判断**：
- 资源如何获得？努力越多越不愿给弱关系
- 资源稀缺？难以再获得的更不愿给

**需求判断**：
- 接受者是慈善？强烈有效需求更可能帮助
- 坏运气vs错误选择？更可能帮助坏运气者
- 有协议？即使无约束力也少有人违反
- 可观察他人行为？看到帮助更可能帮助

**搭便车**：
- 搭便车是受益于他人亲社会行为但不贡献
- 影响相对较小
- 大多数系统可承受少量搭便车
- 搭便车过多时系统崩溃

### 奖励类型

**内在性**：
- 内部：情感"温暖光辉"
- 外部：资源或奖励

**公开性**：
- 私人：只有帮助者看到
- 二元：双方看到
- 公开：所有人看到，激活社会比较和社区规范形成

**时间范围**：
- 短期：即时可见奖励
- 长期：未来不确定日期的奖励

**情境**：
- 一次性：无未来关系的单次互动
- 互惠：未来直接回报

**关系类型**：
- 真实友谊：长期内部奖励，无意图受益
- 交易关系：短期外部奖励，明确意图受益

## 关键洞察总结

### 核心论点

1. **善意游戏是商业机会**：友谊驱动留存，毒性导致流失
2. **设计模式可学习**：双赢互动、牺牲、赠礼经济等
3. **价值观体系化**：必须有、加速器、维护三层结构
4. **过程是园艺**：开发者播种，社区培育

### 设计检查清单

- [ ] 是否设计了双赢互动机制？
- [ ] 是否有前置玩家承诺和行为准则？
- [ ] 是否有牺牲机制创造利他主义机会？
- [ ] 是否避免了稀缺陷阱？
- [ ] 是否有渐进式开放可供性？
- [ ] 是否考虑了安全、相互依赖、共同目标？
- [ ] 是否有健康冲突处理机制？
- [ ] 是否有度量善意（不仅仅是留存）的指标？

### 商业价值

| 指标 | 影响 |
|------|------|
| 留存 | 友谊驱动留存率飙升 |
| 流失 | 毒性导致33%玩家退游 |
| 获客 | 社区口碑是主要渠道 |
| 品牌 | 长期信任积累策略 |
