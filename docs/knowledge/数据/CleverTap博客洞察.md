## 一、RFM 分析模型

**原文**: [RFM Analysis](https://clevertap.com/blog/rfm-analysis/)
**发布日期**: 2025-07-07

### 1.1 核心概念

RFM分析是一种基于三个关键维度的客户细分方法：

| 维度 | 含义 | 衡量指标 |
|------|------|----------|
| **R**ecency | 最近一次购买时间 | 距上次购买的天数 |
| **F**requency | 购买频率 | 特定时期内的购买次数 |
| **M**onetary | 消费金额 | 总消费金额 |

### 1.2 评分体系

使用1-5分的评分系统，可产生最多125个独特细分（5×5×5）：

```
评分标准示例：
┌─────────────────────────────────────────────────────┐
│  Recency: 5分 = 最近7天购买  →  1分 = 超过1年未购买  │
│  Frequency: 5分 = 10次以上/年 →  1分 = 1次/年       │
│  Monetary: 5分 = 消费$500+  →  1分 = 消费<$50      │
└─────────────────────────────────────────────────────┘
```

### 1.3 11种核心客户细分

| 细分类型 | RFM特征 | 描述 | 策略建议 |
|----------|---------|------|----------|
| **Champions** | R↑ F↑ M↑ | 最佳客户，近期频繁高消费 | 奖励忠诚，早期新品体验 |
| **Loyal Customers** | R↑ F↑ M中 | 经常购买，响应促销 | 推荐高价值产品，征集反馈 |
| **Potential Loyalists** | R↑ F中 M中 | 近期购买，有忠诚潜力 | 建立关系，会员计划 |
| **New Customers** | R↑ F低 M中 | 首次购买客户 | 欢迎序列，引导教育 |
| **Promising** | R↑ F低 M低 | 近期购买但金额低 | 免费试用，限时优惠 |
| **Need Attention** | R中 F中 M中 | 活跃度下降 | 个性化推荐，激活活动 |
| **About to Sleep** | R中 F低 M低 | 即将流失 | 价值提醒，重新参与活动 |
| **At Risk** | R低 F↑ M↑ | 曾经优质，现不活跃 | 强力召回，专属优惠 |
| **Can't Lose Them** | R低 F↑ M↑ | 高价值流失客户 | 高端挽留计划 |
| **Hibernating** | R低 F低 M低 | 长期不活跃 | 调研原因，最后激活尝试 |
| **Lost** | R低 F低 M低 | 已流失客户 | 品牌重塑，再营销 |

### 1.4 行业应用场景

```
┌──────────────────┬────────────────────────────────────────┐
│ 行业             │ RFM 应用重点                           │
├──────────────────┼────────────────────────────────────────┤
│ 电商/零售        │ 购买行为分析，个性化推荐               │
│ 金融/银行        │ 交易频率，账户活跃度                   │
│ SaaS/订阅        │ 续费周期，功能使用频率                 │
│ 旅游/酒店        │ 预订频率，消费金额分层                 │
│ 游戏             │ 内购频率，充值金额                     │
└──────────────────┴────────────────────────────────────────┘
```

### 1.5 实施步骤

1. **数据收集**: 整合交易数据，确保数据质量
2. **评分计算**: 为R、F、M分别打分（1-5分）
3. **细分归类**: 根据得分组合划分客户群
4. **策略制定**: 为每个细分设计差异化策略
5. **持续优化**: 定期重新评估，调整阈值

## 二、客户生命周期价值 (CLV/LTV)

**原文**: [Customer Lifetime Value](https://clevertap.com/blog/customer-lifetime-value/)
**发布日期**: 2025-05-13

### 2.1 CLV 计算公式

#### 基础公式

```
CLV = 平均购买价值 × 购买频率 × 客户生命周期长度
```

#### SaaS/订阅模式公式

```
CLV = (ARPU × 毛利率) / 流失率

其中：
- ARPU = 每用户平均收入
- 毛利率 = (收入 - COGS) / 收入
- 流失率 = 月度/年度客户流失比例
```

### 2.2 CLV:CAC 比率基准

| 比率 | 评估 | 含义 |
|------|------|------|
| < 1:1 | 危险 | 获客成本高于客户价值，亏损 |
| 1:1 - 3:1 | 警告 | 有改进空间 |
| **3:1** | 理想 | 行业最佳实践基准 |
| > 5:1 | 可能过低投入 | 考虑增加营销投入 |

### 2.3 14种CLV提升策略

| 策略 | 方法 | 预期效果 |
|------|------|----------|
| **1. 优化Onboarding** | 简化流程，快速价值展示 | 提升激活率 |
| **2. 个性化体验** | 基于行为数据定制内容 | 提升参与度 |
| **3. 客户分层** | RFM细分，差异化服务 | 精准营销 |
| **4. 交叉销售** | 相关产品推荐 | 提升客单价 |
| **5. 向上销售** | 高级版本/套餐推广 | 提升ARPU |
| **6. 忠诚计划** | 积分、会员等级 | 提升留存 |
| **7. 优质客服** | 多渠道支持，快速响应 | 降低流失 |
| **8. 社区建设** | 用户社区，UGC | 增强粘性 |
| **9. 定期互动** | 邮件、推送、活动 | 维持活跃 |
| **10. 反馈闭环** | NPS调研，问题改进 | 提升满意度 |
| **11. 产品迭代** | 基于用户需求开发 | 增加价值 |
| **12. 定价优化** | 动态定价，套餐设计 | 最大化收入 |
| **13. 预流失干预** | 流失预警，主动挽留 | 降低流失率 |
| **14. 转介绍计划** | 推荐奖励机制 | 降低CAC |

### 2.4 真实案例研究

#### MOVii (哥伦比亚金融科技)
- **挑战**: 需要提升用户参与度
- **方案**: 个性化推送，行为触发消息
- **结果**: +327% 消息打开率，+414% 交易转化

#### Boost (电信运营商)
- **挑战**: 高流失率，低参与度
- **方案**: 分层用户策略，智能活动
- **结果**: +70% 留存率提升

#### PayMaya (菲律宾支付)
- **挑战**: 用户激活率低
- **方案**: Onboarding优化，精准推送
- **结果**: +280% 激活率

## 三、留存分析核心指标

### 3.1 关键指标定义

| 指标 | 公式 | 意义 |
|------|------|------|
| **流失率** | (流失客户数 / 期初客户数) × 100% | 客户流失速度 |
| **留存率** | (期末留存客户数 / 期初客户数) × 100% | 客户保持能力 |
| **DAU/MAU** | 日活用户 / 月活用户 | 产品粘性 (理想>20%) |
| **Cohort留存** | 同期群N日/周/月留存率 | 长期健康度 |
| **Drop-off率** | (开始-完成) / 开始 × 100% | 流程优化点 |

### 3.2 Cohort 分析框架

```
同期群留存矩阵示例：

注册周     W0    W1    W2    W3    W4    W5    W6
─────────────────────────────────────────────────
Week 1    100%   40%   25%   18%   15%   12%   10%
Week 2    100%   45%   30%   22%   18%   15%   -
Week 3    100%   50%   35%   28%   22%   -     -
Week 4    100%   55%   40%   32%   -     -     -

→ 横向：产品迭代效果（近期同期群留存提升）
→ 纵向：用户生命周期衰减模式
```

## 四、AI/ML 在营销中的应用

### 4.1 AI Agent 营销架构

```
┌─────────────────────────────────────────────────────┐
│                  AI-First 营销组织                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐          │
│  │ 数据层  │ → │ AI引擎  │ → │ 执行层  │          │
│  │CDP/数据湖│   │ML模型   │   │自动化   │          │
│  └─────────┘   └─────────┘   └─────────┘          │
│       ↑             │             │               │
│       └─────────────┴─────────────┘               │
│              反馈闭环 + 持续学习                    │
└─────────────────────────────────────────────────────┘
```

### 4.2 AI 信任阶梯

| 阶段 | 信任程度 | 应用场景 |
|------|----------|----------|
| **Assist** | 低 | AI辅助决策，人类最终确认 |
| **Augment** | 中 | AI增强人类能力，协作执行 |
| **Automate** | 高 | AI自主执行，人类监督 |
| **Autonomous** | 最高 | AI全自主运行，例外干预 |

### 4.3 Multi-Agent 系统设计

- **协调Agent**: 任务分配与结果整合
- **分析Agent**: 数据处理与洞察生成
- **执行Agent**: 具体任务执行
- **监控Agent**: 系统健康与异常检测
