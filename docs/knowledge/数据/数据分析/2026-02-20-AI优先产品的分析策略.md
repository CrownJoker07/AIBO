## 内容摘要

传统软件产品遵循既定的设计和交互约定：用户通过菜单导航、点击UI按钮、浏览预存信息的屏幕。而AI的采用打破了这些约定，几乎所有数字产品都融入了AI特性。

---

## 一、传统产品 vs AI产品的范式转变

### 传统产品特征

```
┌─────────────────────────────────────────────┐
│           传统软件产品特征                    │
├─────────────────────────────────────────────┤
│  • 离散体验：菜单、按钮、屏幕                │
│  • 确定性交互：点击→响应                     │
│  • 标准转化漏斗：页面→页面→转化              │
│  • 单一"黄金路径"                           │
│  • 预存信息展示                             │
└─────────────────────────────────────────────┘
```

### AI产品特征

```
┌─────────────────────────────────────────────┐
│           AI优先产品特征                     │
├─────────────────────────────────────────────┤
│  • 对话优先界面：文本命令替代点击            │
│  • 概率性响应：相同输入→不同输出            │
│  • 代理式工作流：多任务串行执行              │
│  • 非线性用户旅程                           │
│  • 第三方API依赖                            │
└─────────────────────────────────────────────┘
```

## 二、AI产品分析的三大核心挑战

### 挑战1：成功难以衡量

**问题本质**
- AI工作流的成功不等于特定页面浏览或按钮点击
- 传统分析工具无法将下游指标（收入、留存）直接关联到AI工作流

**解决方案**

| 方法 | 描述 | Amplitude支持 |
|------|------|---------------|
| **Evals** | 客观确定性测试 + LLM-as-a-judge | 事件/属性捕获 |
| **NLP分类** | 对话主题、用例、结果评分 | 自动分类与评分 |
| **定性工具** | 深入分析而非二分法 | Session Replay + Surveys |

**新问题示例**
- 哪些对话主题成功率最高？
- Agent工具使用与用户满意度的关系？
- Token使用量如何影响ROI？

### 挑战2：非确定性输出

**问题本质**
- LLM受多种因素影响：模型选择、系统提示、用户上下文、参数设置
- 输出不可预测，优化空间巨大

**优化维度**

```
┌─────────────────────────────────────────────────────────────┐
│                  AI系统优化维度矩阵                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   输入层           处理层            输出层                 │
│   ────────         ────────          ────────               │
│   • 用户上下文     • 模型选择        • 响应质量             │
│   • 系统提示       • 推理等级        • 延迟                  │
│   • 可用工具       • 温度参数        • 成本                  │
│   • 历史对话       • 批处理策略      • Token使用             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**解决方案**

1. **Feature Experimentation**
   - 远程配置载荷替代硬编码参数
   - A/B测试不同配置的影响

2. **实验问题示例**
   - 哪个系统提示变体带来最高的Agent满意度？
   - 更高推理等级的延迟是否产生更成功的对话？

### 挑战3：外部依赖风险

**问题本质**
- AI功能依赖第三方API（关键路径中的风险）
- 延迟、可用性、成本波动

**解决方案**

| 监控维度 | Amplitude能力 | 行动建议 |
|----------|---------------|----------|
| **延迟** | 自动追踪用户提示到AI响应的时间 | 设置智能警报 |
| **可用性** | API端点状态监控 | Feature Flag熔断 |
| **成本** | Token使用量追踪 | 优化模型选择 |

**熔断器策略**
```
延迟飙升/API不可用 → Feature Flag切换 → 备用基础设施/供应商
```

## 三、用户上下文整合

### 问题
LLM缺乏用户状态和目标的上下文知识：
- 支持聊天机器人不知道用户的付费计划
- 航班预订代理不知道用户的忠诚计划会员身份

### 解决方案：User Profile API

```
┌─────────────────────────────────────────────────────────────┐
│                  数据融合管道                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   CRM ─┐                                                    │
│   CDP ─┼─→ Amplitude User Profile API ─→ 系统提示注入      │
│   DWH ─┤   (行为数据+倾向模型)                              │
│   App ─┘                                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、关键洞察总结

### 传统产品 vs AI产品对比

| 维度 | 传统产品 | AI产品 |
|------|----------|--------|
| 交互模式 | 确定性 | 概率性 |
| 用户路径 | 线性漏斗 | 非线性旅程 |
| 成功指标 | 页面浏览/点击 | Evals + 定性分析 |
| 优化对象 | UI/UX | 模型/提示/参数 |
| 依赖关系 | 内部系统 | 第三方API |

### 核心能力要求

1. **定性+定量融合**：深度理解AI用户体验
2. **实时实验**：快速响应参数、模型、基础设施变化
3. **上下文传递**：将用户数据送达技术栈任何位置
4. **全面监控**：响应性、响应质量、成本的完整可见性
