### CLV 计算公式速查

**来源**: [CleverTap: Customer Engagement](https://clevertap.com/blog/category/customer-engagement/)

```
┌─────────────────────────────────────────────────────┐
│              CLV 计算公式体系                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  基础: CLV = APV × PF × 生命周期                    │
│  SaaS: CLV = (ARPU × 毛利率) / 流失率               │
│  含利润: CLV(利润) = CLV(收入) × 毛利率             │
│                                                     │
│  黄金法则: CLV:CAC = 3:1                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### CLV 三种模型对比

| 模型 | 特点 | 应用场景 |
|------|------|----------|
| **历史CLV** | 回顾性，已实现价值 | 客户细分，Top25%识别 |
| **预测CLV** | 前瞻性，需要假设 | 营销预算分配，高价值客户早期识别 |
| **分段CLV** | 群体平均值 | 渠道价值对比，推荐来源分析 |

### Push vs Pull 互动范式

**来源**: [From Push to Pull](https://clevertap.com/blog/from-push-to-pull-the-next-wave-of-customer-interaction/)

```
Push模式 (品牌主导)        Pull模式 (客户主导)
─────────────────         ─────────────────
潜在客户触达               意图驱动参与
休眠用户唤醒               问题解决/支持
验证活跃用户               流程中即时服务
覆盖非Pull用户             个性化体验

关键: 不是"二选一"，而是"两者兼用"
```

## RFM 客户细分模型

**来源**: [CleverTap: RFM Analysis](https://clevertap.com/blog/rfm-analysis/)

#### 三维评分体系

```
RFM评分 = Recency(1-5) × Frequency(1-5) × Monetary(1-5)
       → 最多125个独特客户细分
```

#### 核心细分策略矩阵

| 细分 | 特征 | 策略 |
|------|------|------|
| Champions | R↑F↑M↑ | 奖励忠诚，新品体验 |
| At Risk | R↓F↑M↑ | 强力召回 |
| Can't Lose Them | R↓F↑M↑ | 高端挽留 |

### 客户生命周期价值 (CLV)

**来源**: [CleverTap: CLV Guide](https://clevertap.com/blog/customer-lifetime-value/)

#### 计算公式

| 模式 | 公式 |
|------|------|
| 通用 | CLV = 平均购买价值 × 购买频率 × 生命周期 |
| SaaS | CLV = (ARPU × 毛利率) / 流失率 |

#### LTV:CAC 黄金法则

```
理想比率: 3:1 (客户价值是获客成本的3倍)
危险信号: < 1:1 (亏损运营)
```

#### CLV提升14策速查

1. 优化Onboarding → 2. 个性化 → 3. 客户分层 → 4. 交叉销售 → 5. 向上销售
6. 忠诚计划 → 7. 优质客服 → 8. 社区建设 → 9. 定期互动 → 10. 反馈闭环
11. 产品迭代 → 12. 定价优化 → 13. 预流失干预 → 14. 转介绍计划

### 留存分析关键指标

**来源**: [CleverTap: Retention Metrics](https://clevertap.com/blog/)

| 指标 | 计算方式 | 健康基准 |
|------|----------|----------|
| DAU/MAU | 日活/月活 | > 20% 为高粘性 |
| 流失率 | 流失数/期初数 | 因行业而异 |
| Cohort留存 | 同期群N日留存 | 看曲线斜率 |

### 领先 vs 滞后指标

**来源**: [CleverTap: Leading vs Lagging Indicators](https://clevertap.com/blog/leading-vs-lagging-indicators/)

| 类型 | 特点 | 示例 |
|------|------|------|
| **领先指标** | 预测性、前瞻性 | 试用数、新注册、页面浏览 |
| **滞后指标** | 结果性、回顾性 | 收入、流失率、利润 |

```
领先指标回答: "我们将去向哪里?"
滞后指标回答: "我们做得怎么样?"
```

### AI-First 产品分析框架

**来源**: [Amplitude: AI-First Product](https://amplitude.com/blog/ai-first-product)

**作者**: Ken Kutyn (Head of Solutions Engineering, APJ, Amplitude)
**发布日期**: 2026-02-20

#### 核心挑战矩阵

| 挑战 | 问题 | 解决方案 |
|------|------|----------|
| 成功难以衡量 | AI工作流无标准转化漏斗 | Evals + Session Replay + Surveys |
| 非确定性输出 | LLM受多种因素影响 | 实验化 + Feature Flags |
| 外部依赖 | 第三方API延迟/可用性风险 | 延迟监控 + 熔断器 |

#### 传统产品 vs AI产品对比

```
传统产品特征              AI产品特征
─────────────────────────────────────────
确定性交互        →      概率性响应
标准转化漏斗      →      多路径用户旅程
页面浏览事件      →      Evals作为事件
单一"黄金路径"    →      非线性用户旅程
定量分析为主      →      定性+定量融合
```

#### 关键洞察

1. **AI界面的概率性特征**导致用户不太可能遵循单一"黄金路径"
2. **传统分析工具难以适应AI产品的动态性**
3. 需要开发**新的指标体系**来评估AI交互质量
4. **现代数字分析平台**需要支持非线性的用户旅程分析

### MCP 集成模式

**来源**: [Amplitude + Lovable](https://amplitude.com/blog/amplitude-lovable-mcp-connector)

**作者**: Anita Mo (Partner Product Marketing Manager, Amplitude)
**发布日期**: 2026-02-23

#### 三阶段闭环

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   数据查询   │ ──→ │   洞察转化   │ ──→ │   发布验证   │
│ (自然语言)  │     │ (建议生成)  │     │ (A/B测试)   │
└─────────────┘     └─────────────┘     └─────────────┘
       ↑                                        │
       └────────────────────────────────────────┘
```

#### 核心价值

1. **无缝集成** - 通过MCP连接器实现直接连接，无需切换工具
2. **实时洞察** - 在构建环境中直接访问用户行为数据
3. **快速迭代** - 发布后立即获取反馈，快速识别并解决问题
4. **降低上下文切换成本** - 提高开发效率

#### 适用场景

- 产品开发团队快速验证假设
- 持续部署和监控
- 用户体验优化
- 数据驱动的产品迭代

#### 核心理念

> **From vibes to signal** - 从"感觉"转向"信号"
> 伟大产品始于直觉，但增长和成熟来自于学习用户实际行为

### 大规模分析治理原则

**来源**: [Zach Phillips 访谈](https://amplitude.com/blog/amplitude-pathfinder-zach-phillips)

**作者**: Beth Saunders (Senior Community Manager, Amplitude)
**发布日期**: 2026-02-25
**更新日期**: 2026-03-11

#### 案例背景

Zach Phillips在Appfire公司从IT人员成长为分析领导者，使用Amplitude管理100+产品。

#### 四大治理原则

1. **标准化先行**: 为100+产品建立统一基准检测标准
2. **文档驱动**: 年度审查和更新流程文档
3. **渐进式采用**: 从简单的页面浏览事件开始，让团队看到价值
4. **回到原始数据**: 不信任表面报告，验证数据来源

#### 关键成功要素

- **数据一致性**: 确保不同产品的数据可比
- **资源分配**: 优先级管理和资源优化
- **团队协作**: 跨团队的数据协作机制
- **持续学习**: 建立学习改进文化

#### 金句摘录

> "一旦他们看到像页面浏览事件这样简单的东西流入Amplitude，灯泡就亮了。"

> "阅读。多少次我以为发现了新问题，结果发现四年前就有文档。"
