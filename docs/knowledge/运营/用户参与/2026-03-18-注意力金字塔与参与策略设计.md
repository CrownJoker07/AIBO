## 核心洞察

> **"当每条消息都假设用户有完全注意力时，用户学会忽略一切。"**

用户的注意力不是 ON/OFF 开关，而是一个连续光谱。他们带着不同深度的注意力到来，受情境、心情和状态影响。大多数参与策略并没有为此设计。

---

## 注意力倒金字塔模型

```
                    ┌─────────────────────┐
                    │    Glance Moments   │  ← 最高频、最低摩擦
                    │    (瞥见时刻)        │
                    │  "我注意到你，但     │
                    │   没有停下来"        │
                    ├─────────────────────┤
                    │  Receptive Moments  │  ← 中等频率
                    │   (接受时刻)         │
                    │   "我意识到，        │
                    │    但很忙"           │
                    ├─────────────────────┤
                    │   Active Moments    │  ← 最低频、最高价值
                    │    (主动时刻)        │
                    │   "我为这个而来"     │
                    └─────────────────────┘
```

### 关键认知

- 大多数应用从**上往下**设计参与策略（从全焦点时刻倒推）
- 更准确的模型是**倒金字塔**：最频繁的用户时刻在顶部
- 转化机会在底部，但大多数用户时刻在顶部
- **品牌如何要求注意力 vs 用户如何拥有注意力** — 这个差距是长期留存崩溃的地方

---

## 三层注意力设计指南

### Layer 1: Glance Moments（瞥见时刻）

**用户心态**："我注意到你 — 但我没有停下来。"

这是最频繁、最容易被低估的时刻。

#### 最佳渠道

| 渠道 | 特点 |
|------|------|
| 锁屏推送 | 零摩擦，无需打开应用 |
| Live Activities | 实时更新，动态信息 |
| Badges | 视觉提示，无声提醒 |
| 轻量级提醒 | 简洁信息传递 |

#### 设计原则

- **零摩擦**：用户不需要做任何事情
- **无要求行动**：不需要点击或响应
- **熟悉、可预测的模式**：用户能预判内容

#### 应用场景

- 比分更新
- 订单状态变化
- 市场波动
- 持续的背景叙事

#### 核心价值

> 这层注意力决定你的品牌随时间感觉**熟悉还是被遗忘**。

当设计得当，瞥见级参与让你的产品在心理上保持"温暖"，而不要求用户做任何回报。

**关键隐喻**：
> 你不能指望一辆一周没启动的车立刻上路。不幸的是，没有流失用户的跳线。我们查过了。

#### 策略：非紧急叙事线程

设计**非紧急叙事线程**，被动地随时间展开。不要把每个锁屏更新视为独立消息，而是一个持续故事的章节：进度、运动或变化。

- 订单缓慢推进
- 余额波动
- 连胜建立
- 系统在用户背后静默工作

当用户能预判他们在瞥见时会看到什么，他们就停止评估是否参与，只是吸收更新。

---

### Layer 2: Receptive Moments（接受时刻）

**用户心态**："我意识到，但我很忙。"

这是日常参与最多的地方，也是**克制最重要**的地方。

#### 最佳渠道

| 渠道 | 特点 |
|------|------|
| 标准推送通知 | 常规消息传递 |
| 消息收件箱 | 应用内消息中心 |
| 微妙的In-App横幅 | 非干扰性提示 |
| Live Activities | 实时表面 |

#### 设计原则

- **情境胜于说服**：提供信息而非推销
- **简洁胜于聪明**：直接明了
- **信息密度无压力**：内容丰富但不强迫行动

#### 应用场景

- 状态更新
- 进度追踪
- 时间敏感意识
- "你可能想知道这个"

#### 核心价值

> 接受时刻是品牌赢得信誉或消耗信誉的地方。这是用户停留时间最长的层级 — **也是最容易过度扩展的层级**。

对这些时刻的克制信号表明你尊重用户的时间和情境，这直接影响选择持久性和消息容忍度。

#### 策略：方向感重于行动

用"方向提示"替代"行动号召"。不要推动用户点击，而是围绕帮助他们理解现在在哪里来构建消息。

从 "Check this out" → "Here's where things stand"

一个简单的转变改变了心理契约。使用这些时刻回答用户可能没意识到的安静问题：
- 一切正常吗？
- 有什么变化吗？
- 我现在需要关心吗？

当消息可靠地回答这些问题而没有压力时，用户变得愿意在真正需要关注时参与。

---

### Layer 3: Active Moments（主动时刻）

**用户心态**："我为这个而来。"

这些时刻**稀有且珍贵**。用户已经有意倾斜进来。

#### 最佳渠道

| 渠道 | 特点 |
|------|------|
| In-App Messages | 应用内全屏或弹窗 |
| 全屏模态框 | 沉浸式体验 |
| 深度链接推送 | 直接跳转到特定功能 |
| Email（已有意图时） | 深度内容沟通 |

#### 设计原则

- **明确价值交换**：用户立即获得什么
- **强叙事或回报**：有意义的内容
- **更少消息，更高赌注**：珍惜每次机会

#### 应用场景

- 账户或政策变更
- 功能发布
- 实时事件
- 高意图生命周期时刻

#### 核心价值

> 主动注意力稀缺，且日益受到用户保护。这些时刻携带最高潜在影响，但只有被**赢得而非假设**时才有效。

过度使用高摩擦消息训练用户抵抗它；谨慎使用使它有意义。

#### 策略：公平的注意力交换

把主动时刻当作明确的价值交换。在部署应用内模态框或深度链接消息之前，问：

**用户立即获得什么作为他们注意力的回报？**

然后使回报明确且单一：
- 一条消息
- 一个想法
- 一个结果

抵抗捆绑更新或堆叠请求。当用户学会全注意力时刻是专注且有价值的，他们就不再防备它们 — 开始有意参与。

## MLB 案例洞察

MLB 理解这个直觉：比分更新不要求点击，投手换人不要求响应。这些时刻让粉丝保持连接而不要求他们**做**任何事情。

> 关系加深即使没有人点击。

**不要错过这个区别**：
- 推送通知不需要点击就算成功
- In-App 消息不总是需要转化
- 有时胜利只是存在，而不是特定警报的表现

如果每条消息都在要求什么，你的用户将学会忽略一切。

## 多渠道策略的本质

多渠道不是关于音量，而是关于**跨越这些层的灵活性**。

能够：
- 耳语（Glance）
- 正常说话（Receptive）
- 在赢得时要求焦点（Active）

用户在这些状态之间不断漂移。你的工作不是把他们推下去，而是**识别他们在哪里并适当响应**。

## 实践清单

### Glance Moments 检查清单
- [ ] 是否零摩擦？
- [ ] 用户需要做任何事情吗？（不应该）
- [ ] 内容可预测吗？
- [ ] 能否在锁屏上一秒理解？

### Receptive Moments 检查清单
- [ ] 是信息还是说服？
- [ ] 是否简洁？
- [ ] 是否尊重用户当前情境？
- [ ] 是否帮助用户保持方向感？

### Active Moments 检查清单
- [ ] 有明确价值交换吗？
- [ ] 只有一个核心信息吗？
- [ ] 这是值得全注意力的时刻吗？
- [ ] 是否稀缺而非频繁？

## 关键引用

> "现代消息的工作不是在每个机会把用户拉回来。而是**在不要求超过时刻能负担的焦点的情况下参与**：瞥见时有意义的更新，帮助保持方向的情境，只有在真正赢得焦点时才有更深度的参与时刻。"

> "耐心、克制和意图是智能手机外持久关系的关键成分。你会惊讶于同样的原则通过屏幕同样适用。"
