```
获取 → 引导(FTUE) → 初期参与 → 首次购买 → 持续参与 → 长期留存 → 再参与
```

### 谜题游戏广告创意策略

#### Match-3 游戏创意要点

| 类型 | 心理驱动 | 视频创意 | 可玩广告 |
|------|----------|----------|----------|
| **Shelf-style** | 日常关联+紧迫感 | 超市/厨房场景，清除→补货循环，叙事性紧迫元素（等待顾客） | 倒计时、密集货架挑战、主题切换测试 |
| **Tile Match** | 放松+简单 | 流畅演示，触觉音效，最短观看时间理解核心机制 | 3-4步快速转化 / 5-6步完整体验，顶部任务目标 |

#### Shelf-style Match-3 进阶策略
- **场景选择**: 超市/厨房/餐厅等**即时可关联**的日常环境
- **节奏设计**: 清除→补货→售罄的持续循环保持**能量感**
- **情感钩子**: 不耐烦顾客、空货架等叙事线索增加**紧迫感**

#### Tile Match 核心原则
> **核心目标**: 展示任何人都能在几秒内开始玩，强调**放松**而非挑战或紧张感

- 快速的"认知热身"定位
- 干净的匹配序列、清晰的视觉效果
- 触觉感的声音效果，强调轻松和满足感

#### Hidden Object 游戏创意要点
- **视频**：快速发现 ↔ 缓慢搜索交替构建张力
- **挑衅设计**：引导手故意犹豫或选错 → 激发"我能找到它！"
- **可玩**："只有4%的人能找到..." 挑战性文案 + 竞争动力
- **长度**：3-5步互动

#### Jigsaw Puzzles 创意要点
- **心理驱动**：美学动机 + 渐进成就感
- **视频**：不完整拼图 + 悬停手提示 → 观众想参与完成
- **轻叙事**：正确放置改善情境 vs 错误放置造成"灾难"
- **可玩**：空棋盘 + 淡背景轮廓引导 → 机制立即可理解
- **注意**：不要在广告中塞入太多元素，避免混乱

#### 可玩广告长度策略
| 长度 | 适用场景 | 效果 |
|------|----------|------|
| 短版 (3-4步) | 即时参与、快速转化 | 立即满足感 → 立即CTA |
| 中版 (5-6步) | 更完整互动体验 | 平衡探索与转化 |
| 长版 (9+步) | 沉浸式体验（拼图类） | 完整场景揭示动画 → 平滑过渡到CTA |

#### 可玩广告设计通用原则
1. **尽早展示价值** - 最小化观看时间理解核心机制
2. **可见的进度** - 屏幕顶部任务目标创造成就感
3. **长度适配受众** - 短版快速转化、长版深度体验
4. **互动即转化** - 每步互动都向CTA推进

### ASO vs 付费UA

| 维度 | ASO | 付费UA |
|------|-----|--------|
| **速度** | 慢（数周/数月） | 快（几小时） |
| **控制力** | 低（依赖算法） | 高（完全可控） |
| **可衡量性** | 有限 | 精确到分 |
| **定位精度** | 广泛搜索 | 精准人群 |
| **CPI** | 技术"免费"（需人力时间） | 真金白银（取决于品类竞争） |
| **灵活性** | 慢（需商店审核） | 快（实时调整） |

### 飞轮效应（ASO + Paid UA协同）
```
Paid UA → 更多安装 → 更高商店排名 → 更多自然流量(ASO) → 更高转化率 → 更低整体CPI
    ↑                                                                      ↓
    ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

**协同机制**:
1. 付费UA推动排名 → 大量安装提升商店排名
2. 排名提升带自然流量 → 高排名带来更多曝光和自然下载
3. 自然流量降低整体CPI → 有机安装不需要广告费
4. 高转化率优化ASO → 更多流量和评论有助于ASO优化
5. 更好的ASO提升付费UA效果 → 优化后的商店页面提高付费流量转化率

### 产品生命周期策略

| 阶段 | 策略重点 |
|------|----------|
| **新应用启动** | 优先Paid UA获取初始用户，同步ASO基础优化 |
| **成长期** | Paid UA和ASO并重，监控自然安装占比变化 |
| **成熟期** | ASO效果显现，可适当降低Paid UA预算，保持最低限度付费投放维持排名 |

### 最佳实践
- **从付费UA开始，用ASO支持**
- 付费UA解决速度和规模问题
- ASO解决效率和长期可见性问题
- 两者结合创造飞轮效应
- **转化率是关键连接点**：优化商店页面让两种策略都更有效

### 动态创意优化（DCO）

#### DCO 核心价值
- **个性化效果**：短期销售提升1.2-7.4倍，长期销售提升1.2-2.7倍
- **用户偏好**：81%的Gen Z和57%的千禧一代偏好个性化广告

#### DCO 工作原理
```
数据输入 → DMP → CMP → 动态组装广告 → 机器学习优化
    │        │      │           │              │
浏览历史/地理位置/设备/天气/CRM
```

#### DCO 六大好处
| 好处 | 说明 |
|------|------|
| 运营效率 | 自动化测试广告组合 |
| 更快洞察 | 快速获得数据驱动洞察 |
| 实时个性化 | 基于用户行为即时响应 |
| 成本效率 | 优化投放，最大化ROI |
| 创意多样性 | 多元素组合测试 |
| 广泛覆盖 | 跨多平台数据整合 |

#### DCO 最佳实践七原则
1. **选择正确创意资产** - 考虑受众偏好，A/B测试
2. **使用多个CTA** - 动态CTA适应不同用户阶段
3. **加大再营销力度** - 顺序消息+动态内容
4. **保持文案简洁** - 紧迫感+社会证明
5. **选择正确宽高比** - 移动信息流优选竖版/方形
6. **简化活动** - 一次测试1-2个变量
7. **快速优化** - 实时监控+预测分析

#### DCO vs 静态创意
| 维度 | DCO | 静态创意 |
|------|-----|----------|
| 适应能力 | 基于数据动态调整 | 固定内容 |
| 优化机制 | AI/ML自动优化 | 仅A/B测试 |
| 个性化 | 实时个性化 | 无 |

#### DCO 设置八步法
```
1. 创建买家画像 → 2. 识别渠道 → 3. 制定定位策略 → 4. 确认消息
       ↓                  ↓                ↓                ↓
5. 连接DMP → 6. 创建决策矩阵 → 7. 结合创意和行动 → 8. 实时优化
```

#### 隐私时代DCO转型
| 挑战 | 应对策略 |
|------|----------|
| 第三方Cookie限制 | 依赖第一方数据 |
| ATT政策限制 | 情境定向技术 |
| 用户追踪减少 | 透明数据实践 |

## 参与度机制设计

### 什么是参与度机制？
专为强化用户行为和提高留存而设计的产品功能，包括：连胜系统、进度系统、应用内货币、社交问责功能、LiveOps活动。

### 参与度机制支持增长的原理
- 提高留存率
- 强化每日习惯
- 增强变现循环
- 用户感受到进步、问责或动力时更可能回归

### 成功案例速查

| 应用 | 类型 | 核心机制 | 增长策略 |
|------|------|----------|----------|
| **AirBuds** | 社交音乐 | 实时分享、每周回顾 | 社交优先设计 → 网络效应 |
| **FlareFlow** | 短剧 | 每日任务、登录奖励 | 复制已验证策略到新品类 |
| **Learna** | AI辅导 | 连胜系统、进度追踪 | 成熟机制 + 新技术 |
| **Pengu** | 社交宠物 | 共同抚养、双货币、LiveOps | 社交问责 × 游戏化 |

### Learna 与 Pengu 深度洞察

#### Learna（AI辅导）- 习惯形成策略
| 策略 | 设计 | 心理学原理 |
|------|------|------------|
| 连胜系统 | 每日+最佳连胜双展示 | 损失厌恶 |
| 进度追踪 | 通话时间柱状图+词汇计数器 | 可感知进步 |
| 即时奖励 | 新用户立即获得"1天连胜" | 早期承诺强化 |

**营销机会**："保持你的连胜！"消息 + 展示可衡量进步

#### Pengu（社交宠物）- 社交问责策略
| 策略 | 设计 | 效果 |
|------|------|------|
| 共同抚养 | 追踪谁喂得更多、谁更关爱 | 脱离=让伙伴失望 |
| 进化系统 | XP升级，解锁新互动 | 长期目标 |
| 双货币 | Coins+Gems，外观+扭蛋 | 深度变现 |
| 功能密度 | 多系统并行（定制/收藏/竞争） | 多受众细分 |

**营销机会**：配对用户获取、推荐策略、按用户类型定制消息（竞争型/收藏家/装饰者/休闲型）

### 核心启示
> **突破性增长往往来自将成熟的参与度机制应用到新的品类，而非发明全新的机制。**

### 参与度机制设计清单

#### 连胜系统（Streak System）
- prominently 展示当前连胜
- 显示历史最佳记录
- 新用户立即获得初始成就感
- 利用损失厌恶心理（断签=打破势头）

#### 进度可视化（Progress Visibility）
- 设定明确的每日目标
- 柱状图展示进度 vs 目标
- 量化努力（时间、数量等）
- 即时反馈

#### 社交问责（Social Accountability）
- 共同任务/目标（如共同抚养）
- 贡献追踪和比较
- 俏皮的竞争提示（"谁是更好的X？"）
- 脱离会影响他人体验 → 强化参与韧性

#### 游戏化系统（Gamification Stack）
- XP 和等级进化
- 双货币经济（Coins + Gems）
- 扭蛋/随机奖励机制
- 深度定制选项
- 季节性 LiveOps 活动
- 内置小游戏

### 注意力金字塔模型（核心框架）

> **核心洞察**：参与度不是 ON/OFF 开关，而是连续光谱

```
                    ┌─────────────────────┐
                    │    Glance Moments   │  ← 最高频、最低摩擦
                    │    (瞥见时刻)        │    锁屏推送、Live Activities、Badges
                    ├─────────────────────┤
                    │  Receptive Moments  │  ← 中等频率
                    │   (接受时刻)         │    标准推送、消息收件箱、In-App横幅
                    ├─────────────────────┤
                    │   Active Moments    │  ← 最低频、最高价值
                    │    (主动时刻)        │    In-App Messages、全屏模态、深度链接
                    └─────────────────────┘
```

#### 三层设计原则

| 层级 | 用户心态 | 设计原则 | 最佳渠道 |
|------|----------|----------|----------|
| **Glance** | "我注意到你，但没有停下来" | 零摩擦、无要求行动、可预测模式 | 锁屏推送、Live Activities、Badges |
| **Receptive** | "我意识到，但我很忙" | 情境胜于说服、简洁胜于聪明、信息密度无压力 | 标准推送、消息收件箱、In-App横幅 |
| **Active** | "我为这个而来" | 明确价值交换、强叙事、更少消息更高赌注 | In-App Messages、全屏模态、深度链接 |

#### 关键认知

> **"当每条消息都假设用户有完全注意力时，用户学会忽略一切。"**

- 品牌如何要求注意力 vs 用户如何拥有注意力 — 这个差距是长期留存崩溃的地方
- 多渠道不是关于音量，而是**跨越这些层的灵活性**
- 用户在这些状态之间不断漂移，工作是识别他们在哪里并适当响应

#### Glance 层策略
- 设计**非紧急叙事线程**，被动地随时间展开
- 当用户能预判他们会看到什么，他们就停止评估是否参与，只是吸收更新
- 这层决定你的品牌随时间感觉**熟悉还是被遗忘**

#### Receptive 层策略
- 用"方向提示"替代"行动号召"
- 从 "Check this out" → "Here's where things stand"
- 回答用户可能没意识到的安静问题：一切正常吗？有什么变化吗？我现在需要关心吗？

#### Active 层策略
- 把主动时刻当作**明确的价值交换**
- 一条消息、一个想法、一个结果
- 抵抗捆绑更新或堆叠请求
