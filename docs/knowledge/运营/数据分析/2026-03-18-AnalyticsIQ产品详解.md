## 产品定位

> **AnalyticsIQ 帮助开发者和工作室理解玩家行为，并将其转化为改善获取、留存和变现的具体行动**

### 核心特性
- 跨平台（移动、Roblox、PC/主机、VR/AR）
- 可扩展（从独立团队到AAA工作室）
- 技术和非技术团队皆可使用
- 实时分析 + 预配置仪表板 + 灵活报告

---

## 解决的核心挑战

### 问题：理解玩家
> 玩家行为复杂。他们因不同原因流失，在不同点转化，根据获取来源、平台或功能曝光而不同花费。

### 传统方法的局限
- 依赖直觉 → 产品决策错误
- 数据有限 → 浪费获取支出
- 缺乏洞察 → 留存差 + 收入损失

### AnalyticsIQ 的解决方案
> **将数据转化为关于玩家的清晰故事，使你能够修复问题、加倍有效做法、自信增长**

## 三大核心用例

### 1. 获取（Acquisition）

#### 核心价值
> **UA支出在上升，每一美元都很重要**

#### 功能矩阵

| 功能 | 说明 |
|------|------|
| **追踪活动影响** | 精确查看哪些广告、创意或网络推动安装，以及这些用户是否真正参与和消费 |
| **超越安装** | 漏斗显示不同活动的玩家如何通过教程、早期关卡和变现步骤 |
| **测量真正ROAS/ROI** | 群组比较按来源的留存、LTV和变现 |

#### 集成的归因平台
- Adjust
- AppsFlyer
- Kochava
- Singular
- Tenjin
- Airbridge

### 2. 参与与留存（Engagement & Retention）

#### 核心认知
> **留存是成功游戏的生命线**

#### 功能矩阵

| 功能 | 说明 |
|------|------|
| **开箱即用KPI监控** | 日留存、会话长度、游戏时间（预构建仪表板） |
| **发现流失点** | 漏斗揭示玩家在教程、关卡或功能中流失的确切位置 |
| **理解功能采用** | 矩阵显示玩家接受哪些功能、忽略哪些 |

#### SegmentIQ 协同
- 玩家分段和个性化
- A/B测试
- 远程配置

### 3. 变现（Monetization）

#### 核心认知
> **收入策略快速变化 — IAP、广告和Webshop都发挥作用**

#### 功能矩阵

| 功能 | 说明 |
|------|------|
| **多模式支持** | IAP、广告变现、直接面向玩家Webshop |
| **广告收入追踪** | 内置支持 AdMob, MAX, ironSource, Fyber, TopOn 等 |
| **IAP分析** | 按物品、ARPPU、付费者行为分析收入 |
| **Webshop贡献** | 查看直接销售如何增加底线 |
| **优化转化旅程** | 漏斗显示玩家在购买前在哪里流失 |

## 核心功能详解

### 1. 实时洞察
- 追踪玩家活动实时发生
- 监控参与、留存和收入
- 查看更新或活动的即时影响

### 2. 自定义仪表板和查询
- 自由探索数据
- 构建反映团队焦点的视图
- 玩家进度、经济平衡、会话流

### 3. 漏斗和群组
- 上下文中理解玩家旅程
- 发现流失点
- 识别驱动长期留存的行为

### 4. 错误和性能监控
- 早期发现技术或游戏问题
- 确保稳定一致的体验

### 5. 作品集级别概览
- 统一查看所有游戏
- 比较指标
- 识别共享优势或风险

## 案例研究：Midnite Games

### 背景
- 知名游戏：Slices（6500万+下载）
- 团队规模：10人
- 需求：快速可靠的玩家行为和游戏健康洞察，但不要数据过载或浪费开发时间

### 解决方案组合
1. **自定义仪表板**
2. **健康监控**
3. **漏斗分析**

### 成果

| 成果 | 说明 |
|------|------|
| **流失触发点识别** | 精确定位玩家流失位置，帮助简化进度流程 |
| **功能转化优化** | 分析不同事件如何驱动参与或购买 |
| **技术摩擦减少** | 通过健康指标追踪崩溃和性能问题（内存、启动时间、错误） |

## 核心洞察

### 1. 猜测成本高
> 决策应由数据支持，而非直觉

### 2. 玩家行为复杂性
- 不同来源 → 不同行为
- 不同平台 → 不同表现
- 不同功能曝光 → 不同结果

### 3. 数据驱动闭环
```
数据 → 洞察 → 行动 → 改进 → 更多数据
```

### 4. 工具整合价值
- 单一平台解决多种需求
- 减少工具碎片化
- 统一数据视图
