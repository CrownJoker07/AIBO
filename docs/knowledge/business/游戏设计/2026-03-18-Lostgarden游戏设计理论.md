## 一、社交系统设计师 (Social Systems Designer)

> 原文: [What is a Social Systems Designer?](https://lostgarden.com/2025/03/22/what-is-a-social-systems-designer/)
> 发布日期: 2025-03-22

### 定义

**社交系统设计师** 定义、构建和平衡决定多玩家如何互动以产生期望社交体验的游戏机械系统。

### 核心关注领域

| 领域 | 说明 |
|------|------|
| **社交物流** | 玩家何时、何地、如何互动？理想的社交密度是多少？ |
| **信任与社交协调** | 如何在从低信任到高信任的个人或群体间定制活动？ |
| **社交资本形成** | 游戏是否在玩家之间建立友谊和社交资本？ |
| **社交规范** | 游戏和更广泛社区的社交价值观和规范化行为是什么？ |
| **病毒性** | 社区如何通过引入新玩家来维持自身？ |
| **毒性生成** | 游戏是否主动产生不良玩家行为和毒性？ |
| **社交经济学** | 驱动期望社交行为的游戏经济平衡是什么？ |
| **社交可供性** | 塑造社交结构的游戏社区和用户流程的 UI/UX 方面是什么？ |

### 为什么现在出现这个角色？

三大趋势的汇聚：

1. **强烈的商业理由**：GaaS 团队发现朋友一起玩是留存的驱动力
2. **经过验证的最佳实践**：积累了需要深入研究和掌握的设计模型和最佳实践
3. **从业者社区**：积极分享和改进的社区

### 职业发展

- 通常是一个高级设计角色，整体上涉及游戏的许多部分
- 很少有人直接被聘入这个角色，而是在需求出现时在特定工作室内部开辟这个空间
- 对于小型开发者，直接将这个角色作为身份，通过实践来推动主流化

### 相关资源

- **概述演示**: 面向学生的社交系统设计演示
- **基本术语**: 社交系统概念写作
- **友谊形成**: 玩家形成深厚友谊的过程
- **信任光谱**: 信任如何支持玩家协调深度
- **人类规模系统**: 人类互动的理想群体规模
- **亲社会游戏设计**: 如何构建预先减少毒性的亲社会结构
- **玩家影响**: 玩家如何影响其他玩家的体验
- **数字繁荣**: 游戏如何帮助玩家繁荣
- **社交可供性**: 限制沟通的选择如何影响社交体验

## 二、善意游戏 (Kind Games)

> 原文: [Kind Games: Designing for Prosocial Multiplayer](https://lostgarden.com/2023/07/08/kind-games-designing-for-prosocial-multiplayer/)
> 发布日期: 2023-07-08
> 工作组成员: Daniel Cook, JC Lau, May Ling Tan, Joel Burgess, Tomo Moriwaki, Erin Drake Kajioka

### 核心定义

**善意游戏** 是从一开始就设计有故意促进亲社会行为系统的多人游戏。

**亲社会行为** 是一个人帮助另一个人的广泛行为类别（独立于可能涉及的意图或奖励）。

### 善意价值观

善意游戏是遵循**善意价值观**的游戏，这些是开发者与玩家签订的顶级契约。

#### 主要行为：亲社会性

玩家**互相帮助**，表现得普遍利他。

#### 支持价值观

| 价值观 | 说明 |
|--------|------|
| **安全** | 玩家感到可以分享和贡献而不必担心恐惧 |
| **相互依赖** | 相互依赖的玩家相互协调，共同实现比个人更多的成就 |
| **共同目标** | 玩家为比自己更大的事情而努力 |
| **归属感** | 玩家属于一个社区，是基本的社交需求 |
| **同理心** | 玩家学会倾听和理解他人的观点 |
| **多样性** | 玩家重视多元身份和观点 |
| **友谊形成** | 玩家与他人形成强烈、有意义的纽带 |
| **健康冲突** | 不可避免的冲突以健康、富有成效的方式处理 |

### 善意游戏定位

```
            帮助他人
                ↑
                │   善意游戏区域
                │   (Sky, Journey, Animal Crossing)
                │
────────────────┼────────────────→ 大群体
                │
                │   竞争游戏区域
                │   (FPS, MOBA)
                ↓
            伤害他人
```

### 典型案例

| 游戏 | 特点 |
|------|------|
| **Sky: Children of Light** | 最佳实践地将新玩家引导进入健康友谊 |
| **MMO 公会和团队副本** | 通过专业角色强调玩家相互依赖 |
| **Animal Crossing** | 仔细管理多人互动以减少毒性 |
| **Journey** | 早期成功的善意游戏示例，几乎不可能恶意破坏 |
| **Elden Ring** | "Let Me Solo Her" 玩家展示了在残酷背景下的利他主义 |

## 三、设计模式与反模式

### 设计模式

#### 1. 让双赢互动变得容易（在限度内）

构建双赢情境，帮助他人变得容易且有回报。

**案例: Overwatch 2 的 Lucio**
- 区域效果能力同时提升队友和自己
- 帮助他人无需零和牺牲

#### 2. 前置玩家承诺和行为准则

大多数玩家使用"以牙还牙"复制策略，只有当期望他人也合作时才会合作。

**关键要素**：
- 明确声明游戏是合作空间
- 定期强化承诺
- 公开示范对不良行为的处理

**案例: Red Dead Redemption 2 的 Rift Trails**
- 马术社区在游戏外组织并安排集体骑行
- 明确的群体身份和期望
- 执行政策

#### 3. 牺牲

在零和交换中，一方的收益导致另一方的损失。但通过适当框架，零和也是通过可见牺牲实现利他主义的机会。

**设计要素**：
- **显示需求**: 另一个玩家通过游戏内可供性发出需求信号
- **可共享资源**: 存在有限的可共享资源
- **显示牺牲**: 帮助者展示他们有能力帮助并放弃资源
- **确认牺牲**: 不外在奖励牺牲玩家，但可以给予可见的社会标记
- **受众**: 朋友或群体内成员更容易触发利他主义

#### 4. 通过对比突出利他主义

如果想让利他行为突出，将其与残酷背景配对。

**案例: Elden Ring 的 Let Me Solo Her**
- 面对极难 Boss Malenia
- 双重选择加入信号真实意图
- 移除交易关系
- 社区价值观的典范

#### 5. 赠礼经济

模仿历史赠礼经济：

**经济设置**：
- 玩家是小型持久群体的长期成员
- 大多数狩猎失败或低于维持水平
- 少数狩猎大获成功（运气成分大）
- 资源对群体所有成员有价值
- 成功是公开知识
- 资源快速衰减

**结果**: 明智策略是赠礼大量资源给群体成员

#### 6. 感知的慷慨

增强感知利他意图的工具：

- **新关系**: 新玩家不知道激励存在
- **合理推诿**: 隐藏激励，使意图模糊
- **价值不对称平衡**: 给予对给予者低成本但对接受者高价值的物品
- **隐藏不对称**: 隐藏财富使赠礼看起来更慷慨

### 反模式

#### 1. 有毒的利他主义

当社交需求过高时，一些玩家付出过多，导致倦怠和流失。

**预防措施**：
- 对齐价值观使工作变得有活力
- 鼓励分布式支持
- 适当的边界

#### 2. 稀缺与丰富

稀缺是强大的经济激励因素，但可能导致毒性。

**毒性稀缺条件**：
- 有限的零和来源
- 共同效用结合嫉妒
- 分享期望
- 陌生人互动

**注意**: 过度丰富也不可取，玩家停止需要他人

#### 3. 自私

自私的人只关心自己的利益。

**自私存在条件**：
- 存在自私/利他行为可能的情境
- 可检测的自私欲望证据
- 理解他人心态的机会和能力
- 利他/自私的期望

#### 4. 社区情境中的个人竞争

过度强调竞争和个人利益会激励自私。

**体育精神的解决方案**：
- 限制竞争空间内的零和资源竞争
- 竞争空间外的共同身份和仪式
- 共同训练和相处时间
- 治愈仪式强调失败中的友谊

#### 5. 黑暗三联征行为

| 特质 | 描述 | 游戏系统风险 |
|------|------|-------------|
| **自恋** | 认为自己最好，值得所有赞扬 | 创造名人或地位的系统 |
| **马基雅维利主义** | 目的证明手段合理 | 奖励游戏结果而非玩家安全 |
| **反社会** | 无悔意和反社会倾向 | 从残酷行为中获益的系统 |

**应对措施**：
- 预测试（马基雅维利主义者和反社会者擅长伪装）
- 权力位置吸引力（考虑反腐设计）
- 回顾性审核（跟踪不良行为模式）

#### 6. 恶意破坏

恶意破坏是社交敌意的一种，目的是限制他人选择。

**案例: Sea of Thieves 的联盟系统**
- 原本设计为合作机制
- 被恶意破坏者用于追踪和骚扰玩家
- 结果：玩家完全避免联盟系统

**教训**：
- 不要做天真的设计师
- 问自己：这个亲社会功能如何被用于作恶？
- 恶意破坏者需要轻松访问非恶意破坏者才能生存

## 四、支持价值观的设计

### 安全

玩家需要能够分享观点和真实自我，而不必担心损害声誉或受到反击。

**设计模式**：

1. **清晰信号**: 当两人进行积极行为时的清晰信号
2. **脚手架开放可供性**: 从最小互动开始，逐步选择加入更丰富的沟通
3. **选择退出并回退到更安全的封闭可供性**: 提供多个同意和控制杠杆

**案例: Sky 的友谊系统**：
- 首先只能作为匿名影子玩
- 交换光揭示持久身份
- 解锁赠送蜡烛能力
- 最终通过选择加入获得开放聊天

### 相互依赖

相互依赖的玩家必须依靠彼此才能实现比个人更多的成就。

**设计模式**：

1. **技能专业化**: 给玩家互补技能和弱点（如 MMO 的坦克/DPS/治疗铁三角）
2. **资源专业化**: 玩家拥有独特资源（如 Animal Crossing 的不同水果）
3. **软合作"在一起更好"**: 他人存在本身带来更好结果
4. **知识共享**: 信息可自由分享，通常对分享者无损失

**反模式: 个人权力**

西方个人主义权力幻想的挑战：
- 玩家升级，获得神一般的能力
- 最终不需要任何人
- 对缺乏现实权力的人有吸引力
- 但在涉及真实人类时变得有毒

### 共同目标

玩家为比自己更大的事情而努力。

**设计模式**：

1. **共同社区目标**: 整个社区共同致力于宏大目标
2. **帮助他人**: 利他目标，特别是涉及帮助或提供他人的长期目标

**案例: Destiny 2 的玩家向导**
- r/DestinySherpa 有 90,000 成员
- 向导帮助玩家通过最具挑战性的内容
- 严格规则禁止自我推广或精英行为

### 归属感

玩家在社区中感到强烈的成员感。

**设计模式**：

1. **加入仪式**: 克服挑战、强调价值观、公开承认成员资格
2. **标记成员资格**: 美学物品展示社会身份
3. **投资和所有权**: 让玩家投入时间、规划和努力

**反模式: 他者化**

煽动对"他者"的焦虑、怨恨或恐惧的策略。

### 同理心

玩家寻求理解他人视角。

**设计模式**：

1. **具身化**: VR 中尝试不同身体和身份
2. **创造反思空间**: 鼓励积极倾听的系统

### 多样性

鼓励多样性和对多样性的容忍。

- 刻板印象导致不真实、交易性对待
- 群体思维导致严重有毒行为
- 需要"超级连接者"建立与其他群体的关系

### 友谊形成

玩家形成强烈、有意义的纽带。

**友谊光谱**: 从陌生人之间的积极互动开始，通过重复的偶然相遇，使用互惠建立更强的信任、同情和支持连接。

**设计模式**：

1. **互惠关系**: 自私动机的行动 → 自动与他人分享 → 低成本互惠 → 友谊升级 → 进一步互惠

**案例: Death Stranding 的社交 Strand 系统**：
- 自私动机：建造桥梁让个人生活更轻松
- 自动分享：你的创作可以出现在其他玩家地图上
- 低成本互惠：其他玩家可以点赞
- 友谊升级：看到更多朋友的作品并互惠

2. **朋友间的戏谑玩笑**: 不是所有越轨行为都是有毒的，信任是关键因素

### 健康冲突

玩家之间的冲突是不可避免的。

**设计模式**：

1. **使用封闭可供性数字化调解冲突**: 识别常见冲突并用 UI 可供性包装，标准化处理流程

2. **决策系统**: 需要覆盖分析、权威、决策机制、沟通渠道、实施、审查/反馈、反腐机制

## 五、制作善意游戏的流程

### 园艺隐喻

多人游戏是在开发者和游戏社区之间的共同创作迭代循环中构建的。

**开发阶段**：

1. **社交架构**: 开发者实现游戏社交系统的基础
2. **播种社区**: 吸引初始社区，玩家承诺和行为准则至关重要
3. **衡量善意**: 超越留存和变现的指标，关注社交网络结构和社交资本
4. **害虫控制**: 处理极端反社会行为
5. **修剪**: 消除或转化不良设计
6. **施肥**: 观察玩家行为，发现共鸣并放大

### 衡量指标

**亲社会指标**：

1. **衡量核心价值**: 为每个关键价值找到代理指标
2. **社交网络分析**: 理解网络形状，友谊和群体分布
3. **心理调查工具**: 测量动机、幸福感的量表

## 六、舒适游戏 vs 善意游戏

### 舒适游戏

关于亲密和反思：
- 关注处理未满足的长期需求
- 出现在个人或小组设置中
- 通常出现在小空间
- 利用对比区分安全空间和严酷空间
- 避免安全环境中的直接压力源

### 善意游戏

关于鼓励善意、利他、亲社会行为：
- 关注我们如何积极相处
- 出现在多人游戏中
- 适用于从二元组到150人的群体设置
- 提供在善意环境中健康调解社交压力源的机制
- 寻求建立信任、友谊和社交资本

### 机会

- **舒适善意**: 能否构建同时包含舒适和善意元素的小规模多人游戏？
- **单机游戏中的善意类比**: 能否在单机游戏中创造帮助玩家建模善意的准社会关系？

## 八、亲社会行为的情境因素

### 纽带强度

最强因素：是否与需要帮助的人有联系。

- 接受者是亲密朋友或亲戚？
- 接受者是特定个人还是一般群体？
- 这是内群体还是外群体？
- 最近是否有互惠？

### 群体规模

Dunbar 数字：一个人可以维持约 150 个关系，但亲密、高信任关系的数量要小得多。

### 价值判断

- 资源如何获得？努力越多，越不可能给弱关系
- 资源稀缺？难以再获得，越不可能给弱关系

### 需求判断

- 接受者是慈善机构？
- 不良情况是由于运气不好还是选择不当？
- 是否与另一方达成协议？

### 搭便车

从他人的亲社会行为中受益而不贡献。

- 大多数社交系统实际上可以承受少量搭便车
- 过多搭便车会导致系统崩溃

## 九、亲社会奖励类型

### 奖励的内在性

- **内部**: 亲社会行为内在激励（如"温暖光芒"）
- **外部**: 获得外部资源或奖励

### 奖励的公开性

- **私人奖励**: 只有帮助者看到
- **二元奖励**: 双方都看到
- **公开奖励**: 每个人都看到，激活社会比较和社区规范形成

### 奖励的时间范围

- **短期奖励**: 即时、高度可见
- **长期奖励**: 未来不确定日期

### 奖励的情境

- **一次性奖励**: 互动发生一次，无未来关系
- **互惠奖励**: 帮助特定个人获得未来直接回报

### 关系类型

- **真实友谊**: 涉及长期内部奖励，双方无意受益
- **交易关系**: 涉及短期外部奖励，每个行为者明确意图使自己受益

## 关键洞察总结

1. **社交系统设计是专业学科**: 需要深入研究社交物流、信任、资本形成等领域
2. **善意游戏是商业机会**: 朋友一起玩是留存驱动力，善意游戏同时满足道德和商业目标
3. **设计模式可学习**: 双赢互动、牺牲、赠礼经济等模式可用于设计亲社会系统
4. **反模式需避免**: 稀缺、自私、个人竞争、黑暗三联征行为会破坏社区
5. **过程是园艺**: 开发者播种基础结构，社区培育社交规范
6. **衡量很重要**: 需要超越留存和变现的指标来衡量善意
