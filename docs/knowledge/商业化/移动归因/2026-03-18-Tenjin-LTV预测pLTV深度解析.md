## 概述

LTV预测是解决混合变现时间差问题的关键技术。Tenjin的pLTV通过神经网络、统一指标和90%准确率解决了传统pLTV模型的挑战。

---

## 什么是LTV预测(pLTV)?

### 定义
- **LTV (Lifetime Value)**: 用户生命周期总收入 = ARPU x 平均生命周期
- **pLTV (Predicted LTV)**: 预测用户生命周期价值，用于早期预测用户价值

### 核心问题：时间差
- 传统LTV需要等待数月才能准确计算
- UA决策需要在D1-D7做出
- pLTV填补这个时间差

---

## 混合变现的pLTV挑战

### 传统pLTV模型的失败原因
| 挑战 | 描述 |
|------|------|
| 数据源分散 | IAA数据来自广告网络，IAP数据来自应用内 |
| 收入延迟 | 广告收入数据24-48小时延迟 |
| 用户行为差异 | 付费用户vs广告观看用户行为模式不同 |
| 预测准确性 | 传统模型在混合变现下准确率低 |

### Tenjin解决方案
| 组件 | 功能 |
|------|------|
| **神经网络** | 学习复杂用户行为模式 |
| **统一指标** | 整合IAA和IAP数据 |
| **90%准确率** | 显著优于传统模型 |

---

## pLTV应用场景

### UA优化
```
pLTV驱动UA决策流程:
1. 获取新用户
2. 收集D1-D3行为数据
3. pLTV预测用户价值
4. 调整渠道出价
5. 优化ROAS
```

### 关键指标对比
| 指标 | 传统方法 | pLTV方法 |
|------|----------|----------|
| 决策时间 | D30-D90 | D1-D7 |
| 数据依赖 | 完整生命周期 | 早期行为特征 |
| 优化周期 | 月级 | 天级 |

---

## 技术实现要点

### 数据整合
- **IAA数据源**: SDK聚合平台 + 广告网络API
- **IAP数据源**: 应用内购买事件
- **行为数据**: 留存、会话、互动

### 预测模型
- 输入: D1-D3用户行为特征
- 输出: 预测LTV (D30/D60/D90)
- 更新频率: 每日
