## 核心案例研究

### 1. Fusee案例：土耳其工作室的3倍增长

**来源**: [How This Turkish Studio Tripled Its Ad Spend After Switching to Tenjin](https://tenjin.com/blog/how-this-turkish-studio-tripled-its-ad-spend-after-switching-to-tenjin-a-fusee-case-study/)
**发布日期**: 2026-01-13

#### 公司背景
- **成立时间**: 2009年
- **专注移动游戏**: 2015年开始
- **累计下载量**: 1.5亿+
- **业务模式**: 混合休闲游戏，2023年开始自发行

#### 成果数据
| 指标 | 结果 |
|------|------|
| UA支出增长 | 3倍 |
| 月收入增长 | 2.5倍 |
| 手动数据挖掘时间 | 0（完全消除） |

#### 挑战分析
1. **高归因成本**: 之前的MMP带来高归因成本和意外隐藏费用
2. **欺诈检测问题**: 广告收入欺诈检测反复出现问题，每周浪费数小时手动检查原始数据
3. **仪表板性能**: 仪表板慢且卡顿
4. **支持延迟**: 支持请求经常被延迟或未解决
5. **SDK依赖**: 设置仅依赖SDK数据，限制了准确性

#### 解决方案
1. **透明定价**: 全包模式，未来功能无额外费用
2. **快速仪表板**: 一致快速可靠的仪表板
3. **专属支持**: 从第一天起通过专用Slack频道提供响应迅速的支持
4. **双源广告收入数据**: 独特的广告收入欺诈检测
   - SDK来自聚合平台的广告收入
   - 直接从广告网络API拉取的广告收入
   - 比较两个数据源快速识别差异和异常

#### 关键洞察
> "高归因成本、意外费用、慢仪表板和不可靠的欺诈检测不仅影响支出效率——它们创造了运营阻力，迫使团队进行手动工作并减慢决策速度。"

## IAA与IAP最佳实践

### 混合变现趋势

**来源**: [IAA & IAP: Differences in Ad Revenue Attribution](https://tenjin.com/blog/iaa-iap-differences-in-ad-revenue-attribution/)
**发布日期**: 2025-12-08

#### IAP特征
| 方面 | 说明 |
|------|------|
| 转化率 | 2%（正常）到5%（非常好） |
| 收入来源 | 直接玩家支出 |
| 玩家基础 | 付费用户，小基数 |
| 游戏设计 | 优化转化 |
| 用户体验 | 付费进度或装饰 |
| 收入可预测性 | 可变，依赖大R |

#### IAA特征
| 方面 | 说明 |
|------|------|
| 收入来源 | 广告商付款 |
| 玩家基础 | 免费游戏，大受众 |
| 游戏设计 | 优化参与度、留存 |
| 用户体验 | 广告支持的免费体验 |
| 收入可预测性 | 随规模更一致 |

#### 混合变现优势
- 跨不同玩家细分最大化收入
- 提供免费访问同时提供高级选项
- 减少对大R的依赖
- 创建更可持续的长期收入流

### 从CPI到价值导向UA

#### 趋势转变
- **过去**: 纯成本每安装(CPI)活动
- **现在**: 价值导向UA活动
- **核心**: 获取更可能成为高价值消费者或与广告互动的用户

#### 关键预测（来自Google专家）
1. **持续增长**: 广告收入将继续增长，可能超过IAP增长
2. **混合变现聚焦**: 更多强调优化混合变现策略
3. **新兴市场机会**: 新兴市场的广告收入增长将更加明显

## 归因与欺诈防护

### 双源广告收入验证

Tenjin的独特优势是提供两种广告收入数据源：

```
┌─────────────────────────────────────────────────────────────┐
│                    广告收入数据源                            │
├─────────────────────────────────────────────────────────────┤
│  来源1: SDK聚合平台                                          │
│  └─ 通过SDK获取的聚合平台广告收入                             │
│                                                              │
│  来源2: 广告网络API                                          │
│  └─ 直接从广告网络API拉取的收入数据                           │
│                                                              │
│  交叉验证 → 识别差异 → 检测异常 → 减少欺诈                    │
└─────────────────────────────────────────────────────────────┘
```

### 定价透明度

| 传统MMP问题 | Tenjin解决方案 |
|-------------|----------------|
| 高归因成本 | 透明、成本效益的定价 |
| 意外隐藏费用 | 全包模式，无额外费用 |
| 未来功能额外收费 | 所有未来功能包含在内 |

## 实践建议

### UA策略优化
1. **从CPI转向ROAS**: 关注用户价值而非安装成本
2. **利用pLTV**: 早期预测用户终身价值
3. **混合渠道策略**: 结合多个广告网络
4. **实时优化**: 使用快速仪表板进行实时决策

### 变现策略
1. **混合模式**: 结合IAA和IAP
2. **广告质量**: 保持高广告质量以保护留存
3. **频率控制**: 合理的广告展示频率
4. **激励视频**: 作为主要广告格式

### 技术基础设施
1. **自动化数据管道**: 减少手动工作
2. **统一仪表板**: 集中所有数据
3. **API集成**: 连接所有广告网络
4. **数据仓库**: 用于高级分析
