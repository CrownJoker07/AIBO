# 应用内广告收入计算框架

> 来源: [How to Calculate Revenue for In-app Advertising: Our Proven Framework](https://www.tenjin.com/blog/)
> 获取日期: 2026-03-18
> 原始发布日期: 2025

## 内容摘要

Tenjin 提供了一套经过验证的 IAA（应用内广告）收入计算框架，解决广告收入数据不一致、难以对账的常见问题。

---

## 核心问题

### 广告收入数据不一致

1. **常见场景**
   - 一个仪表板显示 $50,000 广告收入
   - 另一个仪表板显示 $48,000
   - 广告聚合平台报告一个数字
   - 广告网络报告另一个数字

2. **根本原因**
   - 数据源不同（SDK vs API）
   - 数据延迟差异
   - 汇率换算
   - 统计口径不同

---

## Tenjin 广告收入计算框架

### 双源验证方法

1. **数据源 1：SDK 聚合平台**
   - 来自广告聚合 SDK 的收入数据
   - 实时性高
   - 可能存在延迟

2. **数据源 2：广告网络 API**
   - 来自各广告网络的 API
   - 更准确
   - 通常有 24-48 小时延迟

3. **交叉验证**
   - 对比两个数据源
   - 识别差异和异常
   - 快速检测欺诈

### 收入归因流程

```
广告展示 → 广告网络 → 收入确认
    ↓           ↓           ↓
用户 ID    →  渠道归因  →  收入分配
    ↓           ↓           ↓
UA 渠道   →  ROAS 计算 →  优化决策
```

## 关键指标

### 广告收入核心指标

| 指标 | 定义 | 计算公式 |
|------|------|----------|
| eCPM | 千次展示有效收入 | (总收入 / 展示数) × 1000 |
| ARPDAU | 日活用户平均收入 | 日广告收入 / DAU |
| Fill Rate | 广告填充率 | 填充请求数 / 总请求数 |
| Show Rate | 广告展示率 | 展示数 / 填充数 |

### 归因相关指标

| 指标 | 定义 | 用途 |
|------|------|------|
| Ad ARPU | 广告人均收入 | 评估用户价值 |
| Ad ROAS | 广告 ROAS | 评估渠道效率 |
| Ad LTV | 广告生命周期价值 | 长期价值评估 |

---

## 收入计算最佳实践

1. **数据标准化**
   - 统一时区
   - 统一货币（建议 USD）
   - 统一统计口径

2. **延迟处理**
   - 使用 T+3 数据进行决策
   - 实时数据用于监控
   - 建立数据修正流程

3. **数据验证**
   - 每日对比多源数据
   - 设置异常告警
   - 定期审计数据质量

4. **收入分配**
   - 按用户归因到 UA 渠道
   - 按广告位追踪收入
   - 按广告类型分析效果

---

## 广告收入归因挑战

1. **数据延迟**
   - 广告网络数据 24-48 小时延迟
   - 影响实时 ROAS 计算
   - 需要预测模型补充

2. **多网络整合**
   - 不同网络 API 格式不同
   - 数据更新频率不同
   - 需要统一数据管道

3. **收入分配**
   - 如何将广告收入归因到特定渠道
   - 用户可能来自多个触点
   - 需要归因模型支持

## 实施步骤

### 步骤 1：数据收集

- 集成广告聚合 SDK
- 配置广告网络 API
- 建立数据仓库

### 步骤 2：数据处理

- 数据清洗和标准化
- 延迟数据处理
- 多源数据合并

### 步骤 3：收入归因

- 用户级归因
- 渠道级汇总
- ROAS 计算

### 步骤 4：报告和优化

- 仪表板可视化
- 异常告警
- 优化建议

## 常见问题解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据不一致 | 多源差异 | 统一数据源和口径 |
| 延迟数据 | 网络延迟 | 使用预测和 T+3 数据 |
| 收入归因困难 | 多触点 | 使用归因模型 |
| 欺诈检测 | 虚假流量 | 双源验证和异常检测 |
