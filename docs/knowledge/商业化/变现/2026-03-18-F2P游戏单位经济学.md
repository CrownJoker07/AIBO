# F2P 游戏单位经济学

> 原始发布日期: 2025

## 内容摘要

Tenjin 101 系列采访了斯洛伐克游戏设计师 Michal Tomčány，深入解析 F2P 游戏中最关键但常被误解的概念——单位经济学（Unit Economics）。

---

## 核心洞察

### 单位经济学基础

1. **核心概念**
   - 单位经济学是分析单个用户经济价值的方法
   - 决定 F2P 游戏是否可持续盈利的关键
   - 需要理解 LTV 和 CAC 的关系

2. **为什么重要**
   - 判断商业模式是否可行
   - 指导用户获取预算决策
   - 评估游戏健康度

### LTV (Lifetime Value) 详解

1. **LTV 定义**
   - 用户生命周期内产生的总收入
   - 包括 IAP 收入和 IAA 收入

2. **LTV 计算方法**

```
方法一：简单计算
LTV = ARPU × 平均用户生命周期

方法二：群组分析
LTV = Σ(Day N 收入 / Day 0 用户数)

方法三：预测模型
pLTV = f(早期行为特征) → 预测长期价值
```

3. **影响 LTV 的因素**
   - 留存率（最关键）
   - 付费转化率
   - ARPPU（每付费用户平均收入）
   - 广告展示频次和 eCPM

### CAC (Customer Acquisition Cost) 详解

1. **CAC 定义**
   - 获取单个用户的总成本
   - 包括广告支出和运营成本

2. **CAC 计算**

```
CAC = 总营销支出 / 新用户数量

完整 CAC = (广告支出 + 创意成本 + 运营成本) / 新用户数量
```

3. **降低 CAC 的策略**
   - 优化广告创意
   - 精准受众定向
   - 提高转化率
   - 利用自然流量

### LTV:CAC 比率分析

| 比率 | 状态 | 含义 |
|------|------|------|
| < 1:1 | 亏损 | 每个用户都在亏钱 |
| 1:1 - 3:1 | 边际 | 盈利但增长受限 |
| 3:1 | 健康 | 可持续增长 |
| > 3:1 | 优秀 | 可加大投放 |

### 回收周期 (Payback Period)

1. **定义**
   - 收回用户获取成本所需时间
   - 通常关注 D7、D30、D90、D180 回收

2. **计算方法**

```
回收率 = 累计 LTV / CAC

D30 回收率 = D30 LTV / CAC
```

3. **行业标准**
   - 超休闲：D1-D3 回收
   - 休闲：D7-D30 回收
   - 中重度：D90-D180 回收

---

## 关键行动项

1. **建立 LTV 模型**
   - 收集至少 90 天群组数据
   - 分渠道、分地区计算 LTV
   - 建立预测模型 (pLTV)

2. **优化 CAC**
   - 追踪各渠道真实 CAC
   - 测试不同创意和受众组合
   - 关注有机增长机会

3. **监控关键比率**
   - LTV:CAC > 3:1
   - D30 回收率 > 50%
   - D90 回收率 > 80%

4. **分层分析**
   - 按用户来源分层
   - 按付费行为分层
   - 按地区和设备分层

---

## 常见误区

1. **过度依赖平均值**
   - 平均 LTV 掩盖用户差异
   - 需要分层分析

2. **忽视时间价值**
   - 早期 LTV 不等于最终 LTV
   - 需要长期追踪

3. **低估 CAC**
   - 只计算广告成本
   - 忽略创意和运营成本

4. **混淆收入和利润**
   - LTV 是收入，不是利润
   - 需要考虑分成和成本
