# 大规模多触点归因分析：揭示50%更高的ROAS

> 来源: Singular Blog
> 原文链接: https://www.singular.net/blog/multi-touch-attribution-analysis/
> 原文发布日期: 2026-01-15
> 获取日期: 2026-03-18

---

## 核心摘要

Singular数据科学团队独立分析了**万亿级广告展示和点击**，使用多触点归因(MTA)与最后触点归因进行比较。

### 关键发现

| 指标 | 发现 |
|------|------|
| Meta ROAS | 在MTA下比最后触点归因**高达50%更高** |
| 单一归因安装率 | Meta在移动游戏中达到**94%** |
| 共归因安装率 | Meta仅为**6%** |
| 辅助安装 | Meta生成高达**29%额外辅助安装** |

---

## 归因模型对比

### 最后触点归因 vs 多触点归因

```
最后触点归因回答操作问题：
"谁完成了交易？"

多触点归因回答战略问题：
"谁让交易成为可能？"
```

> **核心洞察**：最后触点告诉你球在哪里越过球门线，多触点归因告诉你它是如何到达那里的。

### 两种模型的角色

| 模型 | 最佳用途 |
|------|----------|
| 最后触点归因 | 计费、合作伙伴问责、日常优化 |
| 多触点归因 | 预算分配、重叠检测、长期增长规划 |

**关键认知**：两者是队友，不是竞争对手。

---

## 为什么发现和社区渠道更依赖多触点归因

### 渠道角色差异

| 渠道类型 | 特点 | 归因表现 |
|----------|------|----------|
| 意图渠道（如付费搜索） | 捕获意图，在转化时刻出现 | 最后触点归因下看起来很强 |
| 发现渠道（如Meta、TikTok、Snap、Pinterest） | 创造意图，早期影响用户 | 最后触点归因下被"低估" |

### 归因视角下的表现

**最后触点归因下**：
- 意图渠道看起来高效
- 发现渠道显得"被低估"

**多触点归因下**：
- 意图渠道仍获得完成功劳
- 发现渠道获得影响、辅助和增量触达功劳

> **核心洞察**：发现渠道很少在终点线看起来最好，因为它们的工作是让用户进入比赛。

## Singular高级辅助框架

### 四种关键指标

1. **单一归因安装**：归因安装没有来自其他渠道的先前触点
2. **共归因安装**：渠道获得最后触点功劳，但另一渠道早期影响了用户
3. **辅助安装**：渠道影响了安装但未获得最后触点功劳
4. **建模MTA指标**：如MTA CPI和ROAS

### 数据分析发现

#### 1. Meta显示最高的单一归因安装率

**94%单一归因安装率**

**这意味着什么**：
- 如果一个渠道持续独自出现在用户旅程中，它更可能在创造需求而非争夺功劳
- 高单一归因率通常表明**增量触达**，不仅仅是高效转化

#### 2. 低共归因表明最小蚕食

**6%共归因安装率**

**这意味着什么**：
- 低共归因意味着增量支出不太可能取代其他渠道创造的价值

#### 3. 辅助率揭示最后触点归因隐藏的价值

**高达29%额外辅助安装**

**这意味着什么**：
- 辅助安装发生在渠道影响安装但未获得最后触点功劳时
- 辅助不是"错过的转化"，而是早期影响发生的证明

#### 4. MTA建模改变ROAS结果

**MTA模型下Meta显示高达50%更高ROAS**

**MTA模型如何工作**：
- 对单一归因安装分配全功
- 对共归因安装分配减少功劳
- 对辅助安装根据可配置权重分配部分功劳

**这意味着什么**：
- MTA下ROAS改善通常表明渠道被低估，而非过度计功

## 营销人员行动指南

### 五条经验法则

1. **不要仅根据最后触点ROAS评估渠道**
2. **使用MTA识别被低估的贡献者**
3. **在扩展支出前监控共归因率**
4. **将高辅助率视为影响信号，而非低效**
5. **配置MTA模型以反映业务目标**

### 实施建议

```
┌─────────────────────────────────────────────────────┐
│           MTA实施决策树                             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  评估渠道绩效                                       │
│       ↓                                             │
│  高单一归因率？──否──→ 检查共归因和重叠            │
│       ↓ 是                                          │
```

**数据规模**：

**研究背景**：作为Meta认证移动测量合作伙伴，Singular独立产出这些发现，与Meta共享以支持测量准确性、透明度和创新的持续合作。

## 关键概念

| 术语 | 定义 |
|------|------|
| 单一归因安装 | 安装没有来自其他渠道的先前触点 |
| 共归因安装 | 渠道获得最后触点功劳，但其他渠道有早期影响 |
| 辅助安装 | 渠道影响安装但未获得最后触点功劳 |
| 增量触达 | 渠道创造的独特触达，非重叠 |
| 发现渠道 | 创造意图的渠道（社交媒体等） |
| 意图渠道 | 捕获现有意图的渠道（搜索等） |

---

## 核心收获

> **"多触点归因不膨胀绩效，它重新分配绩效。"**

- 不同归因方法会实质性地改变绩效解读
- 发现渠道在MTA下展现真实价值
- 高单一归因率表明增量触达
- 高辅助率证明早期影响
- MTA揭示被低估的渠道
