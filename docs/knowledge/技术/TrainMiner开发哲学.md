# 《Train Miner》开发经验：感觉比数据更重要

> 来源: [微信文章](https://mp.weixin.qq.com/s/czbYrnysEUPQduSKda5W3Q) - 制作人分享
> 分类: business / 产品开发
> 标签: 游戏开发、手感优化、留存优化、产品哲学

## 核心观点

> 「有时，感觉比数据更重要」—— Flexus Games 创始人 Semyon Kozyura

## 一、产品背景

- **游戏类型**: 放置游戏 → 混合休闲（放置+经营+模拟解谜）
- **开发周期**: 6个月专注打磨手感
- **商业表现**: 连续近三年每月创造数十万美元营收，仍在稳步增长
- **市场**: 美国为主要投放市场，日均投放广告素材200+组

## 二、关键开发经验

### 1. 寻找原型概念

- **灵感来源**: 团队无系统、持续的灵感碰撞，从主机和PC游戏中寻找
- **判断标准**: 「市面上还没有这样的游戏」的点子才会进入制作
- **案例**: 《Train Miner》灵感部分来自《Loop Hero》的催眠式循环运动感

### 2. 从初测数据平平到继续打磨

| 指标 | 数值 |
|------|------|
| 首测D1留存 | ~35% |
| 美国市场买量回本率 | 3倍 |
| 当前D1留存 | 50% |
| 当前D30留存 | 3.6% |
| 长期玩家平均游玩时长 | 6小时+ |

**关键决策**: 看到美国市场3倍回本率的测试结果后，团队决定继续打磨

### 3. 核心改动：重写代码，重塑手感

- **彻底重构**: 首测后完全重写项目架构，砍掉无用内容
- **操作方式**: 从滑动改为「按住任意位置控制列车前进」
- **物理反馈**: 调整列车装载与空载的物理差异
- **细节打磨**: 改进树木、矿石被切割时的物理反应，加入声音反馈与动态音效变化
- **调整周期**: 整个调整阶段持续六个月

### 4. 观感深度决定留存

- **轨道自动延展**: 让火车轨道随着资源清除而自动延展，为了自然又有反馈感做了大量迭代
- **堆叠高度影响留存**: 把资源堆叠高度调低时，D7留存率瞬跌10%；加入摇晃动画后立竿见影
- **细节增强观感**: 「像这种小细节、能增强观感的东西，玩家非常在意」

### 5. 非传统制作哲学

- **不追求开发速度**: 「如果你逼团队赶进度，就永远做不出有灵魂的游戏」
- **不迷信数据**: 超休闲时代的那套思维已经过时
- **A/B测试但不束缚**: 只在游戏停滞、被补丁拖累时才考虑方向调整
- **敢于止损**: 多次打磨仍无起色则暂停、等待新灵感

### 6. 团队与流程

- **设计师要求**: 必须具备远超平均的感知与审美能力
- **探索型开发者**: 需要在自由状态下爆发创造力的开发者
- **实验阶段**: 不设死线的实验阶段至关重要

## 三、设计细节案例

### 工具系统简化

- **原始方案**: 复杂工具系统（不同资源对应不同工具与耐久度）
- **结果**: 视觉杂乱、节奏混乱，玩家被信息淹没
- **最终方案**: 改为更直观的合并系统

### 内容扩展

| 时期 | 岛屿数量 |
|------|----------|
| 早期版本 | 5个 |
| 当前版本 | 20+个 |

## 四、经验总结

1. **手感是核心竞争力**: 生动有趣的视觉、动效的顺滑反馈、手感的即时响应是长期积累的「爽感」制作功底
2. **数据是信号，不是命令**: D1留存35%时没有砍掉项目，而是继续打磨
3. **细节决定成败**: 堆叠高度调低10%导致D7留存瞬跌10%
4. **团队结构关键**: 需要有审美能力的设计师和探索型开发者
5. **重构要趁早**: 已有作品组合的情况下，早期重构是健康做法
6. **多尝试原型**: 「不要一头栽在一个项目上数年。多做原型、多试错，才有成长」

## 五、对行业的启示

- **观感与手感可以改变游戏命运**: 流畅的交互、细腻的视效、定制动画是破局点
- **难以被复制**: 「很多团队试图克隆《Train Miner》，但他们无法复制那种打磨程度与内部逻辑」
- **高投入拉高门槛**: 这种研发方式虽然拉长周期，但提高了防御壁垒
