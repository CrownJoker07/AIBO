# 马尔可夫模型

> 建模状态转移，预测序列数据

---

## 马尔可夫性质

**未来状态只依赖于当前状态，与过去无关**:

```
P(Xₜ | Xₜ₋₁, Xₜ₋₂, ..., X₀) = P(Xₜ | Xₜ₋₁)
```

---

## 马尔可夫链

### 状态转移矩阵

```
P = | p₁₁  p₁₂  p₁₃ |
    | p₂₁  p₂₂  p₂₃ |
    | p₃₁  p₃₂  p₃₃ |
```

pᵢⱼ = 从状态 i 转移到状态 j 的概率

### 稳态分布

当时间足够长，马尔可夫链会收敛到稳态分布 π：

```
π = π × P
```

## 游戏行业应用

### 用户状态建模

**状态定义**:
- State 0: 新用户
- State 1: 活跃用户
- State 2: 流失用户
- State 3: 付费用户

**转移矩阵示例**:
```
P = | 0.3   0.5   0.1   0.1 |  # 新用户
    | 0.1   0.6   0.2   0.1 |  # 活跃用户
    | 0.0   0.2   0.7   0.1 |  # 流失用户
    | 0.05  0.35  0.1   0.5 |  # 付费用户
```

### LTV 预测

利用马尔可夫链预测用户生命周期价值：
1. 定义用户状态
2. 估计转移矩阵
3. 计算各状态期望停留时间
4. 结合每状态收入计算 LTV

## 隐马尔可夫模型 (HMM)

**状态不可直接观测，需要从可观察序列推断**:

| 组成部分 | 说明 |
|----------|------|
| 隐状态 | 真实但不可见的状态 |
| 观测状态 | 可观察的外显表现 |
| 发射概率 | P(观测\|隐状态) |
| 转移概率 | P(隐状态\|前一隐状态) |

### 游戏应用

**用户意图识别**:
- 隐状态: 浏览、比较、付费意向、放弃
- 观测: 点击序列、页面停留、搜索行为

**用户流失预警**:
- 隐状态: 活跃、沉默、流失
- 观测: 登录频率、功能使用、广告观看

## 马尔可夫决策过程 (MDP)

**加入动作和奖励**:

```
状态 (S) → 动作 (A) → 奖励 (R) → 下一状态 (S')
```

### 核心概念

| 概念 | 定义 |
|------|------|
| 策略 π(a\|s) | 状态 s 下选择动作 a 的概率 |
| 价值函数 V(s) | 状态 s 下的期望累计奖励 |
| Q 函数 Q(s,a) | 状态 s 下执行动作 a 的期望累计奖励 |

### 游戏 AI 应用

- NPC 行为决策
- 资源管理策略
- 动态难度调整

---

## 关键指标

| 指标 | 计算 | 应用 |
|------|------|------|
| 期望停留时间 | 1/(1-pᵢᵢ) | 用户生命周期 |
| 首达时间 | 首次进入状态的时间 | 转化时间预测 |
| 访问频率 | 稳态概率 | 用户分群 |
