# 周报写法指南

> 高效汇报工作进展，让管理者快速掌握团队动态

---

## 周报结构

### 1. 本周进展（Done）

- 用 bullet points 列出完成的关键事项
- 优先展示可衡量的成果
- 标注里程碑达成情况

### 2. 下周计划（Todo）

- 列出预期完成的事项
- 标注优先级和依赖关系
- 预判可能的阻塞点

### 3. 风险与阻塞（Blockers）

- 如实反馈遇到的困难
- 说明需要的支持或决策
- 避免只报问题不给方案

### 4. 学习与反思（Learn）

- 本周学到的新知识
- 需要改进的工作方式
- 对团队的建议

---

## 周报原则

- **简洁**: 控制在 10-15 分钟阅读量
- **具体**: 用数据和结果说话
- **诚实**: 如实反映进展和困难
- **结构化**: 使用统一模板便于追踪

---

## 示例模板

```
## 周报 - [姓名] - [日期]

### ✅ 本周进展
- [完成事项1] - [结果/影响]
- [完成事项2] - [结果/影响]

### 📋 下周计划
- [计划事项1] - [预期结果]
- [计划事项2] - [预期结果]

### ⚠️ 风险与阻塞
- [阻塞点描述] - [需要的支持]

### 💡 学习与反思
- [学到的知识]
- [改进建议]
```
